Exclusion Criteria

 

    

 

For some analysis and display purposes, it can be helpful to eliminate specific elements from consideration, such as credit cards that have been closed as a result of loss or theft.  This capability is supported by the CreditSharp Framework using Exclusion criteria within Analyze and Format operations.  Exclusions can be specified using one of three Boolean flags found in the options classes of Analyze and Format namespaces, or, by using an Exclude property to set individual segments as excluded.

 

There are three predefined conditions by which elements may be excluded:

 

State

Availability

Description

Authorized User account participation

Tradeline

Some credit repair agencies attempt to exploit a loophole in credit reporting practices by arranging for low score borrowers to be added as Authorized Users on good accounts belonging to other individuals.

 

Since Authorized Users do not have any financial obligation or responsibility for the accounts they are authorized to use, they should not gain the benefit of the good credit history.

 

Likewise, they should also not be penalized if the accounts come to have negative credit events or high balances.

 

As such, these type of accounts are often excluded from displays or calculations.

Disputed information

Public Record, Tradeline

If so desired, accounts that are currently being disputed by the consumer can be eliminated from view.  It is up to individual lenders to decide if the dispute is genuine or if the consumer may be trying to 'hide' bad credit while shopping for loans.

Lost or stolen credit cards

Tradeline

These types of accounts are typically excluded from scoring algorithms.

 

The choice to exclude items is indicated either by way of a dedicated Boolean property in an options class, or, directly on the segment to be hidden from view using the data class's Exclude property.

 

AnalyzeOptions: IgnoreAuthorizedUser, AnalyzeOptions.IgnoreDisputed, AnalyzeOptions.IgnoreLostStolen

FormatOptionsHtml: HideAuthorizedUser, FormatOptionsHtml.HideDisputed, FormatOptionsHtml.HideLostStolen

FormatOptionsText: HideAuthorizedUser, FormatOptionsText.HideDisputed, FormatOptionsText.HideLostStolen

FormatOptionsXml: HideAuthorizedUser, FormatOptionsXml.HideDisputed, FormatOptionsXml.HideLostStolen

 

ComIdentification, MrgIdentification: Exclude

ComInquiry, MrgInquiry: Exclude

ComPublicRecord, MrgPublicRecord: Exclude

ComTradeline, MrgTradeline: Exclude

 

The Exclude property of each data class (ComTradeline, for example) displays the current assigned exclude state for each observation and can be used to toggle this value on or off (true or false).  When Exclude is true, the item will essentially be ignored.  Its content will not be included in any analysis operations and will also not appear in any formatted outputs. The CommonData and MergeData objects include a RestoreExcludes method which can be used to turn off all user applied exclusions with a single statement.

 

A use case

 

One possible scenario might involve a business case where all known medical debts are eliminated from any evaluation of a consumer's creditworthiness.  The example shown here demonstrates how this could be accomplished, with each debt being hidden via the Exclude method if they are medical obligations.

 

Note: This state is does not carry on to the MergeData object when applied in the CommonData object.  If items are excluded in a CommonData object, they would still be processed during a merge operation and would appear in the resulting MergeData object.  If items within a MergeData object are to be skipped over, they would need to be excluded directly through that merge instance itself.

 

Example

 

using LuxorTech.CreditSharp.Datamart.Common;

 

namespace HelpDemos

{

  public class HideMedicalDebts

  {

      public void HideItems(CommonData commonData)

      {

          foreach (ComPublicRecord segment in commonData.PublicRecordSegments)

          {

              if (segment.IsMedical)

                   segment.Exclude = true;

          }

 

          foreach (ComTradeline segment in commonData.TradelineSegments)

          {

              if (segment.IsMedical)

                   segment.Exclude = true;

          }

      }

   }

}
