DATE Method

 

    

 

The DATE method is used to obtain an integer equivalent of an element which contains a date.

 

The return value is expressed as the number of months since January 1900.  This value can be used to easily perform date arithmetic or to sort data members in date order.  Some computed values might be:

 

Given date

Integer equivalent

Computed using the following formula: (Date.Year - 1900) * 12 + Date.Month

 

June 14th, 1993

1122

June 14th, 2003

1242

June 14th, 2013

1362

 

If the boolean assumeZero parameter is not provided, then the method will use the value of the AssumeZero property found in the CommonData or MergeData instance being referenced to determine how blank or invalid elements should be treated.

 

When the underlying data element is blank or otherwise invalid and the AssumeZero property, or the local assumeZero parameter, is set to true (the default setting), then the method will return a value of zero (0).  Otherwise an exception would be thrown reporting why a integer value could not be returned.  

 

When the DATE method is being used within a custom attribute definition, any exception is caught internally and the current evaluation is suspended.

 

Syntax

public int DATE (enum T elementId)

public int DATE (enum T elementId, bool assumeZero)

 

Parameters:

 

elementId

Type: enum T (where T is one of the enumerated values specific to the Datamart data class being evaluated)

Indicates the data element to be returned.  (For example: ComTradelineIds.DateOpen)

assumeZero

Type: bool

When true, indicates that any blank or invalid amount values should be treated as if they were zero (0).  This will override the global setting found in the container class's AssumeZero property.

 

Exceptions

Exception class

Status

ParameterException

ParameterStatus.ElementContentInvalid

 

Thrown when the indicated element does not contain date values.

ParameterStatus.ElementIsEmtpy

 

Thrown when the indicated element is blank.

 

Each data class within a CommonData (or MergeData) instance implements its own specific version of the DATE method where the input parameter is an enumerated type specific to the type of data.  For instance, the ComTradelineIds enumeration must be used to evaluate ages within a ComTradeline reference and ComInquiryIds would be used with a ComInquiry instance.

 

Example 1. Specifying sort criteria

 

Setup a custom definition to sort tradelines by open date, in descending order (newest to oldest).

 

using LuxorTech.CreditSharp.Datamart.Common;

using LuxorTech.CreditSharp.Datamart.Merge;

using LuxorTech.CreditSharp.Format;

 

namespace HelpDemos

{

  public class DemoFormat

  {

      public string CreateText(CommonData input)

      {

          FormatOptions options = new FormatOptions(FormatLayout.Text);

          options.Text.Partitions = Partitioning.Report;

          options.Text.ReportStyle = TextStyle.Lending01;

 

          //

          // Obtain a reference for the configuration of an tradeline section in the text layout

          //

          FormatSectionTradline section = (FormatSectionTradline) options.Text.Section("Tradeline");

 

          // Prepare the section for accepting a customized setting

          section.DefineSort("Custom");

 

          // Invert result (multiply by negative one) so that sort is done in descending order

          section.AddSort(x => x.DATE(ComTradelineIds.DateOpen) * -1);

 

          FormatData format = new FormatData(options);

          return format.Generate(input);

      }

   }

}

 

Example 2. Count trades that were open less than two years

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Datamart.Common;

using LuxorTech.CreditSharp.Datamart.Merge;

 

namespace HelpDemos

{

  public class DemoDerivative

  {

      private int ShortTermAccounts(MergeData input)

      {

          int count = 0;

 

          // Instruct operation to throw error if a date is missing

           input.AssumeZero = false;

 

          foreach (MrgTradeline trade in input.TradelineSegments)

          {

              int open = 0;

              int closed = 0;

              try

              {

                   open = trade.DATE(ComTradelineIds.DateOpen);

                   closed = trade.DATE(ComTradelineIds.DateClosed);

              }

 

              // Be sure not to compute when a date is missing

              catch (ParameterException ex)

              {

                  continue;

              }

 

              if (closed - open < 25)

                   count++;

          }

 

          return count;

      }

    }

}

 

Example 3. Alternate count trades that were open less than two years

 

This version of the example above does not rely on catching an exception to avoid blank dates.

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Datamart.Common;

using LuxorTech.CreditSharp.Datamart.Merge;

 

namespace HelpDemos

{

  public class DemoDerivative

  {

      private int ShortTermAccounts(MergeData input)

      {

          int count = 0;

 

          foreach (MrgTradeline trade in input.TradelineSegments)

          {

              if (trade.DateOpen.Length == 0)

                  continue;

              if (trade.DateClosed.Length == 0)

                  continue;

 

              if (trade.DATE(ComTradelineIds.DateClosed) -

                   trade.DATE(ComTradelineIds.DateOpen) < 25)

                   count++;

          }

 

          return count;

      }

    }
