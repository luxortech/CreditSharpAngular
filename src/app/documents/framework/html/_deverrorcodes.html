Exception Codes

 

    

 

Exception codes are reported in the EncodedStatus property of each CreditSharpException or one of its inheritors (BureauDataException, for example).  They are unique numeric values that have several pieces of information encoded within them using bit mapping and can be treated as traditional error codes.

 

There are four distinct elements of information encoded into each error number:

 

Bit field

Description

Width

Domain

Indicates where the error originated.  Distinguishes between an error specific to CreditSharp processing or something thrown by the operating system.

2 bits

Region

Indicates the operational area in which the error occurred, such as Normalization or Formatting.

4 bits

Context

Further refinement of error location.

5 bits

Error code

Reports the specific error condition.

20 bits
