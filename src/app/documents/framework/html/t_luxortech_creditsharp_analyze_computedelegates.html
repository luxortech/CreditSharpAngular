Compute Delegates

 

Namespace: LuxorTech.CreditSharp.Analyze

    

 

The delegates used specifically for evaluating reference instances are the ComputePredicate and ComputeSolution.  A reference instances contain the calculated value of a specific AnalyzeAttribute.  This allows for formulas to be written which use one or more aggregate results.  One way in which this can be useful is for calculating overall percentages.  

 

If, for instance, a business rule is needed for considering the ratio of revolving tradelines to all tradelines, then a Compute solution can be crafted which references two other values: one containing the total number of all tradelines and another containing the total number of revolving tradelines and using those values, calculate the percentage of tradelines that are revolving.

 

The ComputePredicate delegate establishes the inputs and response type for customized logic that determines which inquiries within a credit bureau response will be included in the corresponding calculation.

 

The ComputeSolution delegate establishes the inputs and response type for customized logic that computes a result on specified inquiries.

 

They are used as part of an AnalyzeData implementation that will is comprised of custom attribute definitions.  The delegates can either be inline lambda expressions or references to complied functions/methods which will then be invoked.

 

Predicate Declaration

 

Name

Description

bool ComputePredicate(AnalyzeResult)

References a statement or function which will be evaluated against an existing result instance.

 

Solution Declaration

 

Name

Description

double ComputeSolution(AnalyzeResult)

References a statement or function which computes a numeric value based upon the existing result instance.

 

The following is an example showing how inline lambda expressions can be used to define custom attributes.

Lambda Implementation

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Analyze;

using LuxorTech.CreditSharp.Datamart.Common;

 

namespace HelpDemos

{

    public class PeformAnalysis

   {

        public AnalyzeData Execute(CommonData common)

       {

                 AnalyzeOptions options = new AnalyzeOptions();

                 AnalysisBundle bundle = new AnalysisBundle(AnalysisId.CustomEvaluation, Partitioning.IdMatch, options, "MyCustomAnalysis");

 

                //

                // This example illustrates how to use internal attribute references to compute a value based upon other aggregates.

                // Here, a revolving utilization percentage is computed.

                //

 

                //

                // First define global references to be used for computing Revolving Utilization. Make sure we only include tradelines that are:

                //      A) Revolving, and

                //      B) have a valid CreditLimit or HighCredit

                //

                // The special '@' character indicates the attribute value is to be used for internal calculations only and will not be included in any outputs.

                // Reference attributes can include any aggregated value, not just these internal only values.

                //

                TradelinePredicate isRevWithAmts = (trade, options) => trade.IsRevolving &&

                                                   (!trade.EMPTY(ComTradelineIds.CreditLimitAmount) ||

                                                    !trade.EMPTY(ComTradelineIds.HighCreditAmount) );

 

                TradelineSolution utlizationAmt = (trade, options) => trade.EMPTY(ComTradelineIds.CreditLimitAmount) ?

                                                                      trade.AMT(ComTradelineIds.HighCreditAmount) :

                                                                      trade.AMT(ComTradelineIds.CreditLimitAmount);

 

                bundle.AddEvaluation(new EvaluateTradeline("@TRL_RevBalances", AnalysisOperation.Sum, ComTradelineIds.BalanceAmount, isRevWithAmts));

                bundle.AddEvaluation(new EvaluateTradeline("@TRL_RevLimits", AnalysisOperation.Sum, ComTradelineIds.IdCount, isRevWithAmts, utlizationAmt));

 

                //

                // Now, construct the utilization computation using attribute references.  

                // If the Balance is zero, utilization must be zero, otherwise compute the percentage

                //

                bundle.AddEvaluation(new EvaluateReference("TRL_Utilization", AnalysisOperation.Compute,

                (results) => results.Item("@TRL_RevBalances").Quantity == 0 ? 0 :

                100 * (results.Item("@TRL_RevBalances").Quantity / results.Item("@TRL_RevLimits").Quantity)));

 

                //

                // Gather attributes into a list and compute the values

                //

                List<AnalysisBundle> bundles = new List<AnalysisBundle>();

          bundles.Add(bundle);

          AnalyzeData analyze = new AnalyzeData(bundles);

 

          analyze.Evaluate(common);

          return analyze;

       }

   }

}

 

The example shows the same custom attribute definitions using invokable functions instead, which will typically be the faster implementation.

Function Implementation

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Analyze;

using LuxorTech.CreditSharp.Datamart.Common;

 

namespace HelpDemos

{

    public class PeformAnalysis

   {

        public AnalyzeData Execute(CommonData common)

       {

          AnalyzeOptions options = new AnalyzeOptions();

                AnalysisBundle bundle = new AnalysisBundle(AnalysisId.CustomEvaluation, Partitioning.IdMatch, options, "MyCustomAnalysis");

 

                //

                // This example illustrates how to use internal attribute references to compute a value based upon other aggregates.

                // Here, a revolving utilization percentage is computed.

                //

 

                //

                // First define global references to be used for computing Revolving Utilization. Make sure we only include tradelines that are:

                //      A) Revolving, and

                //      B) have a valid CreditLimit or HighCredit

                //

                // The special '@' character indicates the attribute value is to be used for internal calculations only and will not be included in any outputs.

                // Reference attributes can include any aggregated value, not just these internal only values.

                //

                bundle.AddEvaluation(new EvaluateTradeline("@TRL_RevBalances", AnalysisOperation.Sum, ComTradelineIds.BalanceAmount,

                                                            (TradelinePredicate)REF_IsRevWithAmts));

 

                bundle.AddEvaluation(new EvaluateTradeline("@TRL_RevLimits", AnalysisOperation.Sum, ComTradelineIds.IdCount,

                                                           REF_IsRevWithAmts, REF_UtilizationAmount));

 

                bundle.AddEvaluation(new EvaluateReference("TRL_Utilization", AnalysisOperation.Compute, REF_Utilization));

 

          List<AnalysisBundle> bundles = new List<AnalysisBundle>();

          bundles.Add(bundle);

          AnalyzeData analyze = new AnalyzeData(bundles);

 

          analyze.Evaluate(common);

          return analyze;

       }

 

            private static bool REF_IsRevWithAmts(ComTradeline trade, AnalyzeOptions options)

            {

                if (trade.IsRevolving && (!trade.EMPTY(ComTradelineIds.CreditLimitAmount) || !trade.EMPTY(ComTradelineIds.HighCreditAmount)))

                    return true;

                else

                    return false;

            }

 

            private static double REF_UtilizationAmount(ComTradeline trade, AnalyzeOptions options)

            {

                if (trade.EMPTY(ComTradelineIds.CreditLimitAmount))

                    return trade.AMT(ComTradelineIds.HighCreditAmount);

                else

                    return trade.AMT(ComTradelineIds.CreditLimitAmount);

            }

 

            private static double REF_Utilization(AnalyzeResult results)

            {

                if (results.Item("@TRL_RevBalances").Quantity == 0)

                    return 0;

                else

                    return 100 * (results.Item("@TRL_RevBalances").Quantity / results.Item("@TRL_RevLimits").Quantity);

             }

         }

}
