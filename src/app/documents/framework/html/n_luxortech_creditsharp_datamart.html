LuxorTech.CreditSharp.Datamart Namespace

 

    

 

The Datamart Namespace is where the work of importing and normalizing bureau credit data is performed.  The objects created here provide the foundation for further analysis and content creation in the remaining CreditSharp functional areas.

 

Each credit bureau has its own dedicated namespace (Equifax, Experian or TransUnion) where their bureau data is imported and can be reviewed using the codes and definitions native to that bureau.  The Datamart also has two namespaces for creating and working with normalized data: Common and Merge.

BureauData abstract base class

 

The bureau specific classes are all derived from the same abstract base class - BureauData.  This then allows a bureau specific object to be upcast to the more generic type and implement a have a more streamlined coding pattern, if desired.  For example:

 

      public string GetBureauTtyReport(string bureauName, string creditBuffer)

      {

          string output = string.Empty;

 

          try

          {

              BureauOptions options = new BureauOptions(licenseKey);

              options.DateMask = "MM/01/YYYY";

 

              BureauData bureauData = null;

 

              switch(bureauName)

              {

                  case "efx":

                      bureauData = (BureauData)new EquifaxData(options);

                      break;

                  case "exp":

                      bureauData = (BureauData)new ExperianData(options);

                      break;

                  case "tuc":

                      bureauData = (BureauData)new TransUnionData(options);

                      break;

              }

 

              bureauData.Parse(creditBuffer);

              output = bureauData.Report;

          }

 

          catch (CreditSharpException ex)

          {

               output = ex.Message;

          }

 

          return output;

      }
