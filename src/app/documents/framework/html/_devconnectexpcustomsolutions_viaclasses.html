Using BureauService entries to request custom solutions

 

    

 

The following table illustrates the various Experian features that can be requested via the APIHUB Experian host.  

 

Of the CreditSharp settings indicated below, the ServiceName parameter is the value defined by the CreditSharp framework used to identify the feature being requested.  The ServiceLabel and ServiceId parameters are defined and used by Experian to identify the product or service that is being requested.

 

In Experian nomenclature, the ServiceLabel parameter is known as either the CustomRRKeyword (in NetConnect usage) or the solutionId (in APIHUB usage).  The ServiceId parameter is the unqiue value assigned to each Experian customer for that product or service and is known as either the SolutionId (in NetConnect usage) or the daasReferenceId (in APIHUB usage).

 

If the specification of an Experian scoring model is also required when ordering one of these custom features, use the AddModel method to include that detail.
