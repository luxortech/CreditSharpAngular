ComAlert Class

 

Namespace: LuxorTech.CreditSharp.Datamart.Common

    

 

Defines an instance of a common bureau alert segment.

Normalized elements

BureauCode

Text (6)

Gets the original code returned by the given bureau.

 

The values presented are dependent upon which bureau and which service provided the information.  The message text property will contain the fully articulated description of this coded value.

 

This element may be left blank and will never be popuated when the ServiceId is 'File'.

 

MessageText

Text

Gets a more detailed description of the BureauCode value.

MessageType

Char (5)

Gets a value indicating what type of alert is being reported.

 

Values:

Alert = The item is a warning that should be addressed or researched carefully.

Check = From Experian only, offers supplemental identification for consumer.

Error = Indicates the bureau encountered an error while attempting to fulfill the requested service.

Note = The item is informational in nature.

Status = Provides the outcome of a given request, primarily for indicating why a credit profile was not returned (such as an inquiry that yields a "No Record" response for a consumer with no credit history).

XtAncestry

Text

Gets the details showing what bureau segment(s) where used in creating the current instance.

 

The value returned is comprised of four separate numeric elements, delimited by colons.  For example: 1:2:14:5

 

The first position reports the BureauLocale from which the segment originated.  The second position indicates which Bureau reported the segment.  The third position identifies which kind of bureau segment was used to create the segment.  Finally, the fourth position indicates which sequential occurrence within the bureau response is responsible for the current item.

 

In the example above, this would indicate the item in question originated in the United States locale (1), from the Experian credit bureau (2), is an inquiry segment (14), and was the fifth inquiry in the response data.

 

The values in the third position are taken from the each bureau's segment id enumerations (Equifax segment ids, Experian segment ids or TransUnion segment ids) so would be translated dependent upon which bureau was indicated in the second position.

 

Lastly, when looking at a segment created from multiple sources (a merged record), there will be more than one set of Ancestry fields which will themselves be delimited by a semi-colon, as in:

 

1:2:24:4;1:4:87:13

 

This would indicate that the current tradeline is a merged result from the fourth Experian record and the thirteenth TransUnion record.
