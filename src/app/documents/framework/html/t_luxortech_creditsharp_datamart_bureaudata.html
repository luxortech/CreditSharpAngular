BureauData Class

 

Namespace: LuxorTech.CreditSharp.Datamart

 

    

 

The BureauData type is an abstract base class which defines the generic properties and methods for importing credit bureau data.  Each of the three bureau specific types for processing a bureau response (EquifaxData, ExperianData, and TransUnionData) are derived from this class, which can be convenient when trying to consolidate code by upcasting the bureau specific object to the more generic base type.

 

For example:

 

      public string GetBureauTtyReport(string bureauName, string creditBuffer)

      {

          string output = string.Empty;

 

          try

          {

              BureauOptions options = new BureauOptions(licenseKey);

              options.DateMask = "MM/01/YYYY";

 

              BureauData bureauData = null;

 

              switch(bureauName)

              {

                  case "efx":

                      bureauData = (BureauData)new EquifaxData(options);

                      break;

                  case "exp":

                      bureauData = (BureauData)new ExperianData(options);

                      break;

                  case "tuc":

                      bureauData = (BureauData)new TransUnionData(options);

                      break;

              }

 

              bureauData.Parse(creditBuffer);

              output = bureauData.Report;

          }

 

          catch (CreditSharpException ex)

          {

               output = ex.Message;

          }

 

          return output;

      }

 

The references below outline the members and features that can be accessed through this generic base class.  As there is no public constructor for this class, it can only be used as a target for upcasting and streamlining bureau processing, as demonstrated above.

 

Name

Description

Methods

AddModel

Adds a new custom model definition for later reference when normalizing the bureau data.

Image

Returns a byte array which contains a PDF credit report image supplied by the current credit bureau.

Parse

Performs the work of extracting all individual elements (segments, fields, reports) from a credit bureau response. Refer to the bureau specific implementation of this method for more details.

Properties

Buffer

Gets the content of the originating credit bureau data.

Buffer2

Gets the only the machine readable content of the originating credit bureau data (suppresses any eye-readable/printable portions).

BureauId

Gets the enumeration specifying which credit bureau is represented.

BureauLocale

Gets the value indicating the country from which data was obtained.

FilteredView

Gets the value indicating if the bureau data is being viewed in its native format.

Layout

Gets the value indicating the layout found within the provided credit bureau input buffer.

Report

Gets the printable, pre-formatted credit bureau report if one was provided in the response.

Segments

Gets a collection of references to segments extracted during the Parse() operation.

Version

Gets the value indicating the version of the underlying bureau data.
