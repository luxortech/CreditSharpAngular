AddSort Method

 

Namespace: LuxorTech.CreditSharp.Format

    

 

The AddSort method is used to define custom criteria for ordering members in an output group or section.

 

Sort criteria are defined by either using lambda expression as the input argument or providing a delegate to a custom sort function.  What is being identified, in either form, is the particular value on which an item is to be sorted.  If the sorted result needs to be determined using multiple elements, then separate sort criteria should be added for each individual element, with the least significant criteria being supplied first and the most significant being supplied (added) last.

 

The underlying sort algorithms will always sequence data in ascending order (smaller numbers and older dates will appear first, strings will be in alphabetic order, false boolean values appear before true values).  If a descending order is desired, then the sort criteria must be inverted to achieve the desired result, which is discussed further in the Remarks below.

 

To ensure consitent sorting of dates and numeric values consider using the Derivative helper methods when establishing sort criteria.  Without using a derivative method, the sort operation will be carried out on the character or string based representation each data element, meaning the output will be arranged alphabetically rather than numerically.  The Remarks section below also provides more detail on this implementation.

 

Syntax

 

public void AddSort (Func <T, object> sortCriteria)

 

Parameters:

 

sortCriteria

Type: function delegate [typically provided as a lambda expression] where T is a reference to the CreditSharp objects being sorted (ComAddress, ComTradeline, etc).

 

The delegate function should accept a reference to a CreditSharp instance and return the extracted data element on which to make a sort decision (typecasted as an object).

 

Exceptions

 

Exception class

Status

FormatDataException

FormatDataStatus.InvalidOperation


ParameterException

ParameterStatus.ElementOutOfRange

 

Remarks

 

Sort criteria is specified one element at a time and can go on to include as many elements as needed.  If sorting against multiple elements, the least most significant item criteria should be added first and the most significant added last.

 

Compound sort criteria

For example, to order tradelines alphabetically by Account Name, and then with that by Date Open and within that by Balance one would use the following instructions:

 

            FormatSectionTradeline tradeSection = (FormatSectionTradeline)options.Html.Section("Tradeline");

           tradeSection.DefineSort("Custom");

           tradeSection.AddSort(x => x.AMT(ComTradelineIds.BalanceAmount));

           tradeSection.AddSort(x => x.DATE(ComTradelineIds.DateOpen));

           tradeSection.AddSort(x => x.AccountName);

Numeric and descending criteria using Derivative methods

Each of the examples above makes use of derivative helper methods to achieve the desired outcomes.  All reference to balance amounts is performed using the AMT method and the date open is likewise filtered through the DATE method.  

 

Using these methods also offers the easiest way to sort items in descending order.  The underlying code always sorts elements in ascending order so values must be logically inverted to have values returned in descending order.  This can be achieved by multiplying numeric or date values by negative one or using the INV method to reverse character strings.  Here are the same three examples from above, with everything inverted to be in descending order:

 

            FormatSectionTradeline tradeSection = (FormatSectionTradeline)options.Html.Section("Tradeline");

           tradeSection.DefineSort("Custom");

           tradeSection.AddSort(x => x.AMT(ComTradelineIds.BalanceAmount) * -1);

           tradeSection.AddSort(x => x.DATE(ComTradelineIds.DateOpen) * -1);

           tradeSection.AddSort(x => x.INV(AccountName));

The first example illustrates setting up a custom sort for the Address section of a Text output while the second illustrates a custom Tradeline section for an Html output.  Examples specific to other output sections can be reviewed within the Derived Classes topic in this chapter.

 

Example using lambda expressions

 

using System;

 

using LuxorTech.CreditSharp.Analyze;

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Datamart.Common;

using LuxorTech.CreditSharp.Datamart.Merge;

using LuxorTech.CreditSharp.Format;

 

namespace HelpDemos

{

  public class DemoFormat

  {

      public string CreateText(MergeData input, AnalyzeData aggregates)

      {

          FormatOptions options = new FormatOptions(FormatLayout.Text);

          options.Text.Partitions = Partitioning.Report;

          options.Text.ReportStyle = TextStyle.Lending01;

          options.Text.ShowSummary = true;

 

          //

          // Obtain a reference for the configuration of an address section in the text layout

          //

          FormatSectionAddress addressSection = (FormatSectionAddress) options.Text.Section("Address");

          addressSection.Verbose = true;

 

          // Prepare the section for accepting a customized setting

          addressSection.DefineGroups("CUSTOM");

 

          FormatGroupAddress addressGroup = addressSection.AddGroup("Current");

          //

          // Establish the selection criteria for the first group, using a lambda expression

          //

          addressGroup.Selector = (x => x.IsCurrentAddress == true);

          addressGroup.InitialHeader = "Current address" + Environment.NewLine +

                                      "---------------";

          //

          // Establish the selection criteria for the second group, using a lambda expression

          //

          // Best practice is to ensure that selections can only appear in one group

          //

          addressGroup = addressSection.AddGroup("Former");

          addressGroup.Selector = (x => x.IsCurrentAddress == false);

          addressGroup.InitialHeader = "Former address" + Environment.NewLine +

                                        "--------------";

          // Now, set up a custom sort order

          addressSection.DefineSort("CUSTOM");

          //

          // Use the DATE helper function to simplify date comparisons and invert the value (multiply by

          // negative one) so that items are returned in descending order (most recent dates first).

          //

          addressSection.AddSort(x => x.DATE(ComAddressIds.FirstReported) * -1);

          addressSection.AddSort(x => x.DATE(ComAddressIds.LastReported) * -1);

 

          FormatData format = new FormatData(options);

          return format.Generate(input, aggregates);

      }

   }

}

 

Example using delegate functions

 

using System;

 

using LuxorTech.CreditSharp.Analyze;

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Datamart.Common;

using LuxorTech.CreditSharp.Datamart.Merge;

using LuxorTech.CreditSharp.Format;

 

namespace HelpDemos

{

  public class DemoFormat

  {

      public string CreateText(MergeData input, AnalyzeData aggregates)

      {

          FormatOptions options = new FormatOptionsHtml();

          options.Layout = FormatLayout.Html;

          options.Html.Partitions = Partitioning.Report;

          options.Html.ReportStyle = HtmlStyle.Common01;

          options.Html.TradelineStyle = HtmlTradelineStyle.Tradeline04;

 

          //

          // Obtain a reference for the configuration of an tradeline section in the html layout

          //

          FormatSectionTradeline tradeSection = (FormatSectionTradeline)options.Html.Section("Tradeline");

          tradeSection.Verbose = true;

 

          // Prepare the section for accepting a customized setting

          tradeSection.DefineGroups("CUSTOM");

 

          //

          // Establish the selection criteria for each account type group

          //

          FormatGroupTradeline tradeGroup = tradeSection.AddGroup("Col");

 

          tradeGroup.InitialHeader = "COLLECTION ACCOUNTS";

          tradeGroup.MissingHeader = "COLLECTION ACCOUNTS: <span class=\"red\">None found</span>";

 

          tradeGroup.Selector = ChooseCollections;

 

          tradeGroup = tradeSection.AddGroup("Mtg");

          tradeGroup.InitialHeader = "MORTGAGE ACCOUNTS";

          tradeGroup.MissingHeader = "MORTGAGE ACCOUNTS: <span class=\"red\">None found</span>";

 

          tradeGroup.Selector = ChooseMortgage;

 

          tradeGroup = tradeSection.AddGroup("Ins");

          tradeGroup.InitialHeader = "INSTALLMENT ACCOUNTS";

          tradeGroup.MissingHeader = "INSTALLMENT ACCOUNTS: <span class=\"red\">None found</span>";

 

          tradeGroup.Selector = ChooseInstallment;

 

          tradeGroup = tradeSection.AddGroup("Rev");

          tradeGroup.InitialHeader = "REVOLVING ACCOUNTS";

          tradeGroup.MissingHeader = "REVOLVING ACCOUNTS: <span class=\"red\">None found</span>";

 

          tradeGroup.Selector = ChooseRevolving;

 

          // Now, set up a custom sort, Order by: AccountName, BalanceAmount (descending)

           tradeSection.DefineSort("CUSTOM");

          tradeSection.AddSort(GetSortAmount);

          tradeSection.AddSort(GetSortName);

 

          FormatData format = new FormatData(options);

          return format.Generate(input, aggregates);

      }

 

      private bool ChooseCollections(ComTradeline trade)

      {

          return trade.IsCollection;

      }

 

      private bool ChooseInstallment(ComTradeline trade)

      {

          return trade.IsInstallment;

      }

 

      private bool ChooseMortgage(ComTradeline trade)

      {

          return trade.IsMortgage;

      }

 

      private bool ChooseRevolving(ComTradeline trade)

      {

          if (trade.IsRevolving == true ||

               trade.IsOpen30Day == true ||

               trade.IsCreditLine == true)

              return true;

          else

              return false;

      }

 

      //

      // Use the AMT helper function to simplify amount comparisons and invert the value (multiply by

      // negative one) so that items are returned in descending order (highest balances first).

      //

      private object GetSortAmount(ComTradeline trade)

      {

          int date = trade.AMT(ComTradelineIds.BalanceAmount) * -1;

          return (object)date;

      }

 

      private object GetSortName(ComTradeline trade)

      {

          return (object)(trade.AccountName);

      }

   }

}
