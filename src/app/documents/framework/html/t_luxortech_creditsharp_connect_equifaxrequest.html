EquifaxRequest Class

 

Namespace: LuxorTech.CreditSharp.Connect

    

 

The EquifaxRequest class is used to prepare and execute an Equifax request for credit data.  This class is used to transmit requests to the Equifax system and capture the responses for later reference.  In addition to transmitting the request, the class also allows for the retrieval the actual inquiry string to aid in record keeping or troubleshooting.

 

The AddModel method is where requests for any bureau scoring models would be created and configured.

Example

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Connect;

 

    public class EquifaxConnect

    {

        public string OrderJoint(Applicant primary, Applicant secondary)

        {

            BureauConnection connection = new BureauConnection(BureauIds.Equifax);

           connection.HttpSettings.ServiceName = "acrotest";

           connection.HttpSettings.ServiceURL = "https://transport5.ec.equifax.com/servlet/stspost";

             connection.HttpSettings.WebUserId = "myCompanyId";

             connection.HttpSettings.WebPassword = "letMeIn";

 

            EfxInquiryOptions options = new EfxInquiryOptions(licenseKey);

           options.ConsumerReferral = true;

           options.MilitaryBorrowerCheck = true;

 

            EquifaxRequest request = new EquifaxRequest(options);

           request.PermissiblePurposeCode = "01";

           request.ProductCode = EfxProductCode.CreditReport;

           request.ResponseVersion = EfxVersionNumber.Version50;

           request.SubscriberCode = "222XX11111";

           request.SubscriberPassword = "ABC";

           request.CpuResponse = EfxCpuResponse.FullFileFixed;

 

            string creditData = request.RequestProduct(connection, primary, secondary);

 

            return creditData;

       }

   }

Locale

Enum

One of the BureauLocales enumerations.

Gets or sets the value indicating if the request is being submitted for a Canadian or US resident.

 

Equifax will inspect the state/province code in a given address and automatically return bureau data from the corresponding national repository.  For instances where an application contains an address in each country, Equifax will dutifully include a response from each locale.  

 

CreditSharp cannot currently accept responses that include both national repositories in a single request, so this enumeration is used to filter out whichever address does not match the indicated locale.

PdfStatus

Num (3)

Gets the numeric value indicating whether or not an attempt to automatically download a PDF image of a consumer credit report from the new cloud based API was successful.

 

* When the value is -1, it indicates no attempt was made to obtain a PDF report.

* When the value is 0, it indicates the PDF was successfully obtained as has been embedded within the JSON response.

* When the value is greater than zero, it indicates the attempt to obtain the PDF report failed.

 

If the PdfStatus indicates a failed attempt, the value will contain an error code from one of two possible .NET enumerations. Please refer to the error tables below for the range of values.

 

In the advent of a failed attempt, the PDF file(s) will need to downloaded directly at a later time.  They are preserved in the Equifax system for up to two hours following the initiating inquiry.  Please refer to the PDF Reports topic in this section for details on downloading PDF content with a subsequent request.
