Mop and Payment History Options

 

    

 

The appearance of Html layouts tradeline payment patterns and graphical statuses can be controlled using several different options:

 

Setting

Description

TradelineStatusField (accessed via FormatOptionsHtml)

Controls if status shading is based upon the Current MOP or the Maximum MOP

UseAlphaCurrentMop (accessed via FormatOptions)

Indicates if current months within the payment pattern should be shown as 'C' or the standard '0' and '1'.

UseSatisfiedMop (accessed via FormatOptions)

If true, indicates if zero balance collection/charge off/repossession accounts be shown as Satisfied rather than delinquent.

ZeroBasedMop (accessed via FormatOptionsHtml)

Bases payment pattern values either from '0' or '1'.  

 

When true, current months are shown as '0', 30 day lates are '1', 60 days lates are '2', etc.

 

When false, current months are '1', 30 day lates are '2', 60 day lates are '3', etc.

 

In the examples below, note that the BK OF AMER tradeline has a current mop of 'STS' due to the UseSatisfiedMop setting being true.  Also, in the first example, the graphic status display appears green because the TradelineStatusField is looking at the CurrentStatus (ie: current Mop) while in the second it is red because it is looking at the MaximumStatus (ie: maximum Mop).
