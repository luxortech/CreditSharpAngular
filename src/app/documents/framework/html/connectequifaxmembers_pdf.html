PDF Reports

 

    

 

With the introduction of the new cloud-based API, Equifax has also modernized their printable eye-readable credit report layout.  The traditional text-based reports (Acrofile Plus, etc) have been deprecated in favor of a new graphically based PDF report.  The PDF report can be included with a standard credit data request or can be ordered on it's own.

 

The reports are not returned directly in response to an inquiry, as was the case with the legacy system's text based reports.  Instead, a uniquely named PDF report is created and saved on the Equifax server and a URL link(s) is returned in the response.  If the inquiry is a Joint request, then a URL link will be provided for a PDF report on each consumer.  The URL link(s) must be used to then download the actual PDF content, which is preserved in the Equifax system for up to two hours.

 

This URL can be used to directly communicate with the Equifax server to obtain the PDF report.  Even with the URL however, this essentially requires a wholly separate transaction that must be authenticated as if you were ordering any other credit report.  This separate transaction can be performed in your own code outside of the CreditSharp domain, but perhaps the most convenient method is to have CreditSharp do that work instead, by issuing a second request and providing the URL as an optional inquiry input.  The topic "Retrieving a PDF image from Equifax" covers this scenario.

 

Alternatively, you can elect to have the PDF content automatically download and embedded it within the actual JSON credit data response. An option can be enabled when submitting a request which instructs CreditSharp to immediately parse the credit data response, extracting the URL links and automatically submitting the request to download the PDF content.  The graphical PDF data is then inserted into the credit data as a base64 encoded string.  The topic "Requesting a PDF image from Equifax" covers this scenario.
