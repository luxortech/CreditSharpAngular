MrgBanking Class

 

Namespace: LuxorTech.CreditSharp.Datamart.Merge

    

 

Defines an instance of a merged credit banking segment.

 

In practice, these items are not actually merged, simply copied directly from the input.

Amount

Text (15)

The amount being reported.  This may be a record of the balance on the account, or depending upon the Status Code, the amount of an NSF check.

 

Amounts may be expressed as approximations, so the value presented here may not always be strictly numeric.  Approximated values are identified with the keyword LOW, MED or HI.  These are then followed by a numeric value indicating how many digits exist in the amount.

 

The LOW, MED and HI keywords are used to indicate what range the initial digit occupies, where LOW means 1, 2 or 3, MED means 4, 5, or 6 and HI means 7, 8 or 9.

 

For example, a value of 'MED 5' would indicate the amount was a five digit number whose first digit was a 4, 5 or 6.  This would equate to a possible range from $40,000 to $69,999.

DtSequence

XtAncestry

Text

 

The value returned is comprised of four separate numeric elements, delimited by colons.  For example: 1:2:14:5

 

The first position reports the BureauLocale from which the segment originated.  The second position indicates which Bureau reported the segment.  The third position identifies which kind of bureau segment was used to create the segment.  Finally, the fourth position indicates which sequential occurrence within the bureau response is responsible for the current item.

 

In the example above, this would indicate the item in question originated in the United States locale (1), from the Experian credit bureau (2), is an inquiry segment (14), and was the fifth inquiry in the response data.

 

The values in the third position are taken from the each bureau's segment id enumerations (Equifax segment ids, Experian segment ids or TransUnion segment ids) so would be translated dependent upon which bureau was indicated in the second position.

 

Lastly, when looking at a segment created from multiple sources (a merged record), there will be more than one set of Ancestry fields which will themselves be delimited by a semi-colon, as in:

 

1:2:24:4;1:4:87:13

 

This would indicate that the current tradeline is a merged result from the fourth Experian record and the thirteenth TransUnion record.

XtSources

List

Gets a list of all the original segments which were merged into the current instance.

 

The list will be of the type from which the current segment is derived.  For example, the XtSources property within the MrgAddress class will have the ComAddress references listed which were used to create the current merged reference.
