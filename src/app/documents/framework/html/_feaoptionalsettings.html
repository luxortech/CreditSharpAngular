In addition to accessing and assigning the various options through these class instances, settings can also be applied by deploying a configuration file to the same folder as the CreditSharp assembly.  Details about using this configuration file can be found in the CreditSharp.dll.config topic.  The configuration file can also be used to define the name and adverse factor text for custom scoring models.

Beginning with BureauOptions in the table above and moving downwards, there are two settings which are seen in each component:  DateMask and MaskSsn.

 

In practice, these would only ever really need to be employed when using the FormatData component to generate an output of some sort, as this is typically the only instance where data is physically extracted from the CreditSharp framework.

 

However, if your system requirements will see you interacting directly with the data classes exposed within one of the Datamart namespaces (EquifaxData, ExperianData, TransUnionData, CommonData or MergeData), these option settings can be explicitly stated during object creation to ensure that field level access to the relevant elements will reflect the desired masking.

 

A description of these two options follows.

Option name

Content

Description

DateMask

Text (10)

Controls how date fields are displayed and what should value be assumed when the day portion of a date is missing.

 

The input will consist of a string with placeholders for where each date portion should appear in output fields, along with the desired delimiter (if any) desired between elements.

 

Given the input bureau date of March 13th, 2023, the example below show what each possible input mask would yield:

 

Input mask

Displayed output

"MM/01/YYYY"

= 03/13/2023

"YYYY/MM/01"

= 2023/03/13

"YYYYMM01"

= 20230313

"MM-01-YYYY"

= 03-13-2023

"MM-01-YY"

= 03-13-23

 

As can be seen above, the Day portion of the input mask has been specified using a numeric two character value.  This not only serves to position the day portion of the date in the output string(s), but also provide a default value to be used when a Day element is not returned with a bureau date.

 

For example, given a bureau date of simply July, 2023, the following outputs can be generated:

 

"MM/15/YYYY"

= 07/15/2023   (A default day of 15 has been applied)

"YYYY/MM/01"

= 2023/07/01   (The more typical default day of '01' has been applied)

 

The DateMask property is initialized as: "MM/01/YYYY".

 

MaskSsn

Char (1)

Will mask (overwrite) the first five digits of all social security numbers with the provided character.  This can be a number, a letter, or any other printable character.

 

Given a social security number of 666751234:

 

MaskSsn = "9" would yield: "999991234"

MaskSsn = "*" would yield: "*****1234"

MaskSsn = "x" would yield: "xxxxx1234"
