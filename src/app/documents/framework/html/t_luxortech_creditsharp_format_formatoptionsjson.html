FormatOptionsJson Class

 

Namespace: LuxorTech.CreditSharp.Format

    

 

The members found within the FormatOptionsJson class provide access to settings for controlling the creation of JSON based outputs.

 

 

Methods

 

Name

Description

FirstSection

Method establishes the first section to be displayed in the output.

NextSection

Method adds another section to the growing collection of items to be displayed in the output.

Section

Provides access to configure how and what sections are displayed in the output.

Description

Decorate

Text

Gets or sets the value to be used for creating a 'pretty' JSON output.

 

If the value is set to an empty string, then no whitespace characters will be inserted into the JSON stream.  Otherwise, new line characters will be used to terminate each JSON node and the supplied value will be used to indent each nested line.  The input can be either the tab character or spaces.

 

[Defaults to '\t']

ShowBooleanProperties

Bool

Gets or sets the flag indicating if boolean properties should be displayed.

 

Certain segments include extended boolean properties reporting whether or not specific data conditions exist in each observation. The segments which have these properties are: ComIdentification, ComInquiry, ComPublicRecord and ComTradeline. One such example is the IsMedical property which indicates if the underlying debt is due to medical expenses.

ShowSources

Bool

Gets or sets the flag indicating if item sources should be included in the output.

When enabled, a Source object will be emitted to display the bureau, ident and match values to document from which bureau the item originated and to which specific consumer the item belongs.

 

The ident value corresponds to the XtIdentNumber property of a given segment and the match value corresponds to the XtIdentMatch property.

ShowTradeLates

Bool

Gets or sets the flag indicating if each tradeline delinquency show be shown in separate elements.

 

The Delinquencies property if the ComTradeline segment records the specific date and severity of ever known delinquency. This information is represented in condensed for within the payment history pattern but can also be explicitly documented with a separate node for each delinquent occurrence using this optionally property setting.

 

[Defaults to false]

Verbose

Bool

Gets or sets the flag indicating if coded elements should be displayed with their corresponding descriptive text.

 

The setting works in conjunction with the Verbose property found in each FormatSection. When this property is true, then the section level property will be checked to see if its coded members should include descriptive text.  In this way, it is possible to include descriptive elements on a section by section basis.

 

This most common use of this feature is to include descriptive text in just the Score elements so that factor code text is shown, while not consuming unneeded space by having potentially redundant verbose elements everywhere else within the JSON stream.

 

[Defaults to false]

ZeroBasedMop

Bool

Get or sets the value which determines if a Tradeline Manner of Payment and Payment History codes will be numbered from zero or one.

 

When true, an instance of a current month would be represented as "0" while a 30 day late would be seen as "1", a 60 day late as "2", etc.  When false, the current month would be styled as "1" with 30 day lates being seen as "2" and 60 day lates being "3", etc.  Having the codes zero based allows for the explicit representation of a 180 day delinquent occurrence, as "6".  When the codes are based from one, then the value of "6" must stand in for either 150 or 180 days late.

 

Please refer to the Mop Codes appendix topic for a full presentation of this optional feature.

 

[Defaults to false]
