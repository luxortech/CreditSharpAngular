EvaluationEnable Method

 

Namespace: LuxorTech.CreditSharp.Analyze

    

 

The EvaluationEnable method is used to re-enable one or more attributes that may have been disabled.  The input parameter can identify a single attribute or use a special wildcard syntax to identify a group of attributes.

 

For example, "AUT025" would affect just that one attribute, whereas "AUT*" would affect all attributes whose names begin with AUT, while "AUT4*" would affect only those which begin with the AUT4 string.

 

Syntax

public int EvaluationEnable (string attributeName)

 

Parameters:

 

attributeName

Type: string

The name of the attribute(s) to be disabled.

 

Example

 

      public void AnalayzeTRL(CommonData common)

      {

          AnalyzeOptions options = new AnalyzeOptions();

          options.IgnoreLostStolen = true;   // Do not count trades reported lost or stolen

 

          List<AnalysisBundle> bundles = new List<AnalysisBundle>();

          AnalysisBundle bundle = new AnalysisBundle(AnalysisId.AttributeFactory01, Partitioning.IdMatch, options);

 

          // First, disable all evaluations

          bundle.EvaluationDisable("*");

          // Now, enable just the TRL attribute group

          bundle.EvaluationEnable("TRL*");

 

          bundles.Add(bundle);

 

          AnalyzeData analyze = new AnalyzeData(bundles);

          analyze.Evaluate(common);

      }
