FormatSection Class

 

Namespace: LuxorTech.CreditSharp.Format

    

 

The FormatSection base class can be used to review or modify settings for individual display sections within the various CreditSharp Format outputs.  The Expanded and Visible properties both control how sections are displayed while the Verbose property can be used to trigger additional output detail.

 

A series of data specific classes are derived from this base class providing additional customizations for manipulating how elements are sorted and/or grouped within each section.  The easiest method for accessing a section's base properties is directly through the Section method found in each layout's options class, for instance:

 

options.Html.Section("Directory").Visible = false;

 

 

Base Properties

 

Name

Content

Description

Expanded

Bool

Indicates if the initial state of the section is collapsed or expanded (used with Html outputs only).

 

[Defaults to true]

Name

Text

Gets the Name of the current section.

Verbose

Bool

Indicates if the current section should include supplemental detail.

 

With Html and Text layouts, the property indicates if sections with no members should still be displayed.  If enabled (true) then the section header will be printed along with a "None found" notation, otherwise the section will not appear at all.

 

With Json and Xml layouts, the property indicates if additional descriptive text should be generated to accompany enumerated or coded elements.

 

[Defaults to false]

Visible

Bool

Indicates if the current section should or should not be displayed.

 

In addition to the shared properties listed above, a derived class exists for each specific type of display section.  There is a separate and distinct class associated with each section of a CreditSharp layout.  In each case, the FormatSection base class is extended to include methods, specific to each section, that can be used to customize or configure grouping and sorting criteria for a given section.  

 

Extended Methods

 

Name

Description

AddGroup

Adds an instance of a specific class which can be used to define grouping criteria for the current section.

 

Grouping can only be employed within Html and Text layouts.

AddSort

Adds a custom sort criteria for ordering members within the current section.

DefineGroups

Assigns a pre-defined group definition to the current section, or,  prepares the reference to receive custom grouping criteria.

DefineSort

Assigns a pre-defined sort definition to the current section, or,  prepares the reference to receive custom sorting criteria.

Group

Retrieves a reference to a data specific FormatGroup instance previously added to the section.

 

The various data specific classes derived from the FormatSection base class each implements its own version of the methods listed above. The links provided below offer details on what sort and group configurations are currently defined for use within the CreditSharp framework, along with specific examples for implementing customized definitions.
