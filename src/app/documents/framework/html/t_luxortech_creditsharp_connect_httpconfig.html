HttpConfig Class

 

Namespace: LuxorTech.CreditSharp.Connect

    

 

The HttpConfig type exposes properties required to submit transactions over an internet connection using Http protocols.  The class is made available through the HttpSettings property found in each bureau's request object.

 

Example

 

      public string OrderViaProxy(Applicant primary, Applicant secondary)

      {

          BureauConnection connection = new BureauConnection(BureauIds.TransUnion);

          connection.HttpSettings.ServiceURL = "https://netaccess-test.transunion.com";

          connection.HttpSettings.WebUserId = "myCompanyId";

          connection.HttpSettings.WebPassword = "notPassword";

 

          connection.HttpSettings.CertificateStore = "Trusted Root";

          connection.HttpSettings.CertificateLocation = "CurrentUser";

          connection.HttpSettings.CertificateName = "TUNA Cert";

 

          connection.ProxySettings.Mode = ProxyType.Configured;

          connection.ProxySettings.HostId = "127.0.18.101";   // To be replaced with actual setting

          connection.ProxySettings.PortNbr = 8080;           // To be replaced with actual setting

 

          TucInquiryOptions options = new TucInquiryOptions(licenseKey);

          options.RequestRegion = TucRequestRegion.ProductionSystem;

 

          TransUnionRequest request = new TransUnionRequest(options);

          request.PermissiblePurposeCode = "AU";

          request.ProductCode = TucProductCode.CreditReport;

          request.ResponseVersion = TucVersionNumber.Version40;

          request.SubscriberCode = "1234567";

          request.SubscriberPassword = "A999";

          request.SubscriberMarket = "06";

          request.SubscriberSubMarket = "22";

          request.TtyResponse = TucTtyResponse.ContinuousSheet;

 

          BureauScore bureauScore = request.AddModel("00R82");

          bureauScore.SupplementalData = ScoreSupplement.RangeAndRank;

 

          string creditData = string.Empty;

          if (secondary != null)

                creditData = request.RequestProduct(connection, primary, secondary);

          else

               creditData = request.RequestProduct(connection, primary);

 

          return creditData;

       }

 

 

Properties

 

Name

Content

Description

Please refer to the expandable sections below for details on what to provide for each of the three credit bureaus

 

AuthorizationId

Text

Gets or sets the id value to used to authenticate next generation server access conforming to the OAuth standard.

AuthorizationSecret

Text

Gets or sets the passcode value to used to authenticate next generation server access conforming to the OAuth standard.

CertificateLocation

Char (12)

Gets or sets the value indicating in which system account a required client side certificate resides.

CertificateName

Text

Gets or sets the friendly name associated with the desired client side certificate.

CertificateStore

Char (20)

Gets or sets the value indicating where a required client side certificate is located.

ServiceName

Text

Gets or sets the value indicating what service is being accessed at the host site.

ServiceURL

Text

Gets or sets the URL which hosts a given bureau's credit bureau communication interface.

WebPassword

Text

Gets or sets the password paired with the WebUserId.

WebUserId

Text

Gets or sets the bureau assigned user id specifically used to authenticate internet access.

 

Equifax settings

Property

Usage

AuthorizationId

Corresponds to the OAuth2 client_id assigned by Equifax for use only when connecting to the new cloud-based API.

AuthorizationSecret

Corresponds to the OAuth2 client_secret assigned by Equifax for use only when connecting to the new cloud-based API.

CertificateLocation

not used with Equifax

CertificateName

not used with Equifax

CertificateStore

not used with Equifax

ServiceName

When connecting to the legacy front-end, use "acrotest" when testing and "acro" when getting live consumer data.

When connecting to the cloud-based front-end use "api".

ServiceURL

When connecting the legacy front-end, use:

https://transport5.ec.equifax.com/servlet/stspost

 

When connecting the cloud-based front-end for testing purposes, use:

https://api.uat.equifax.com/v2/oauth/token

 

When connecting the cloud-based front-end for obtaining live consumer data, use:

https://api.equifax.com/v2/oauth/token

WebPassword

When connecting to the legacy front-end, this equates to the password supplied in the authorization header.

 

When connecting to the cloud-based front-end, this is the password associated with the OAuth2 credentials.

WebUserId

When connecting to the legacy front-end, this value is typically known as the site_id.

 

When connecting to the cloud-based front-end, this is the username associated with the OAuth2 credentials.
