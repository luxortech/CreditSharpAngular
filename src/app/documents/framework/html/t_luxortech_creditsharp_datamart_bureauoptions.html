BureauOptions Class

 

Namespace: LuxorTech.CreditSharp.Datamart

    

 

The BureauOptions class provides a container for invoking any desired optional features and passing those settings on to the constructor of a bureau processing object (EquifaxData, ExperianData, and TransUnionData).

DateMask

Text (10)

Provides a date mask string indicating the order and positioning of Month, Day, Year when displayed along with the default day value.

 

The input will consist of a string with placeholders for where each date portion should appear in output fields, along with the desired delimiter (if any) desired between elements.

 

Given the input bureau date of March 13th, 2023, the example below show what each possible input mask would yield:

 

"MM/01/YYYY"

= 03/13/2023

"YYYY/MM/01"

= 2023/03/13

"YYYYMM01"

= 20230313

"MM-01-YYYY"

= 03-13-2023

"MM-01-YY"

= 03-13-23

 

As can be seen above, the Day portion of the input mask has been specified using a numeric two character value.  This not only serves to position the day portion of the date in the output string(s), but also provide a default value to be used when a Day element is not returned with a bureau date.

 

For example, given a bureau date of simply July, 2023, the following outputs can be generated:

 

"MM/15/YYYY"

= 07/15/2023   (A default day of 15 has been applied)

"YYYY/MM/01"

= 2023/07/01   (The more typical default day of '01' has been applied)

 

The DateMask property is initialized as: "MM/01/YYYY".

 

MaskSsn

Text (1)

Gets or sets the character to be used as a mask for Social Security numbers.

 

If the property is left blank (the default), all social security data will appear unmasked, as it was received from the credit bureau.  Otherwise, the character provided here will replace the first five positions of each SSN field.

 

options.MaskSsn = "*";

 

With this setting, the SSN 666-99-6060 would be output as *****6060, or with dashes, ***-**-6060 if on a printed report.

 

Any printable character can be supplied here.
