LuxorTech.CreditSharp.Extract Namespace

 

    

 

The Extract Namespace is where users can elect to export credit bureau information which adheres to each bureau's native definitions.  The names of data elements correspond, for the most part, to those found within the bureau's own manuals.  As well, all the codes and content is presented using the bureau specific conventions.

 

The output can be generated using either Json or Xml structures.  Json and/or Xml schemas for each bureau can be downloaded from the CreditSharp website at www.CreditSharp.com/public/documentation.

 

By default, all data presented in an extract will have been cleansed so that dates and numeric or dollar amounts are properly and consistently formatted.  These values are not normalized (converted to common or shared definitions) but they are also not in the exact image as returned by the credit bureau.  If there is a need to review the original raw data in its uncleansed form, setting the FilteredView property of the BureauData class will achieve that goal.  Review the FilteredView discussion below for an example of this feature.

 

An example of a typical programming pattern

 

using System.Diagnostics;

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Datamart;

using LuxorTech.CreditSharp.Datamart.Equifax;

using LuxorTech.CreditSharp.Datamart.Experian;

using LuxorTech.CreditSharp.Datamart.TransUnion;

using LuxorTech.CreditSharp.Extract;

 

namespace HelpDemos

{

  public class DemoExtract

  {

      public string GetBureauXml(string creditDataBuffer, BureauIds creditBureau, string licenseKey)

      {

          BureauData bureauData = null;

          string output = string.Empty;

 

          switch(creditBureau)

          {

              case BureauIds.Equifax:

                  bureauData = new EquifaxData(new BureauOptions(licenseKey));

                  break;

              case BureauIds.Experian:

                  bureauData = new ExperianData(new BureauOptions(licenseKey));

                  break;

              case BureauIds.TransUnion:

                  bureauData = new TransUnionData(new BureauOptions(licenseKey));

                  break;

          }

 

          try

          {

               bureauData.Parse(creditDataBuffer);

 

              ExtractData extractData = new ExtractData(new ExtractOptions(ExtractLayout.Xml));

              output = extractData.Assemble(bureauData);

          }

 

          catch (CreditSharpException ex)

          {

              Debug.WriteLine(LuxorTech.CreditSharp.Core.Utilities.ErrorStack(ex));

          }

 

          return output;

      }

   }

}

Using the FilteredView property.

 

The short coding sample below illustrates setting the FilteredView property instructing the framework to return unfiltered (or raw) bureau data.

 

using LuxorTech.CreditSharp.Datamart;

using LuxorTech.CreditSharp.Extract;

 

namespace HelpDemos

{

  public class DemoExtract

   {

      public string GetBureauJson(BureauData bureauData)

       {

          ExtractData extractData = new ExtractData(new ExtractOptions(ExtractLayout.Json));

           bureauData.FilteredView = false;

 

          return extractData.Assemble(bureauData);

       }

   }

}
