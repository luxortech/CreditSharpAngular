Working with data: missing and numeric content

 

    

 

The following topics highlight key points to consider when crafting custom data evaluations.

 

By default, all credit data elements exposed within the CreditSharp framework are presented as character strings.  This is necessary in order to handle situations where some values that are typically numerics or dates can be seen to consist of non-numeric values.  Using strings also allows for recognizing the distinction between an amount that was explicitly reported as 0 versus one that was left blank.

 

In light of this, when performing analysis on various credit data elements, care must be taken to anticipate how blank or missing data can affect computations.  Each condition is discussed in the following topics.  To see how it all fits together, please visit the Computing revolving credit utilization topic.

 

Working with Blank or Missing credit data values

 

AssumeZero option

 

Frequently, dollar amounts (and some numeric amounts) are left blank in the credit bureau responses.  Both the CommonData and MergeData classes include an AssumeZero property which indicates how the CreditSharp framework resolves this data condition.  By default, the AssumeZero property is set to true meaning all blank/missing values will be treated as zero.

DefaultEmptyCalcs option

 

Another point to consider is how attributes that could not logically be calculated should be exhibited.  Consider an attribute that attempts to add up the total available credit could be accessed using a consumer's Home Equity Credit Lines.  The consider the consumer who does not have any HEQ credit lines.  

 

If the reported attribute value defaults to zero it may mistakenly communicate that the consumer has already reached their limit on HEQ credit lines, even though they don't actually have any HEQ credit lines.  Leaving the attribute value blank can help alleviate this possible confusion.

 

The DefaultEmptyCalcs property found in the AnalyzeOptions class controls this presentation, with the default setting being true.

 

Working with numbers, dates and amounts

 

Each Common or Merge credit data class has helper methods which can be used to ensure that comparisons and calculations which reference number, date, or amount elements yield desired results. These helper methods take care of obtaining the underlying numeric value of a given data element when performing any type of custom evaluation.

 

In conjunction with the AssumeZero property discussed above, number or amount fields are treated as zero if they are blank (or alphabetic) when the appropriate wrapper method is used to obtain the value.  If the AssumeZero property is set to false then any evaluation which encounters a non-numeric value (blank or alphabetic) will be suspended.  As mentioned above in the Average balance example, this effectively skips or ignores the active instance.

 

The four helper methods used for this purpose, AMT(), NBR(), DATE() and EMPTY(), are fully outlined within the Derivative Methods topic.  The amount, number and date methods can be invoked using either the globally assigned AssumeZero property or can be provided with a local parameter to selectively override the global setting.
