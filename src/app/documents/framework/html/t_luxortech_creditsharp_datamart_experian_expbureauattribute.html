ExpBureauAttribute Class

 

Namespace: LuxorTech.CreditSharp.Datamart.Experian

    

 

Defines an instance of the CreditSharp extension ExpBureauAttribute class.

 

This class aids in the processing each of the Experian services which deliver attributes by automatically splitting the all-in-one response strings into multiple separate instances of an attribute name/value pair.

 

Properties

 

Name

Description

Native data elements

Indicator

Indicates the origin or status, as reported by the Experian service.

Name

The attribute name or id.

 

Typically, content length will be a minimum of six characters and a maximum of thirty, depending on which service generated the data.  If the service was Instant Prescreen or Prequalification, the names are taken from the user defined values in the ExperianData.DecisionSchema property.

Sign

Indicates if the value is a positive or negative number

Source

Indicates which service has reported the current attribute.  This corresponds to the ARF response's segment id from which each instance was extracted.

 

Values:

 

'361' = Instant Prescreen or Prequalification results

'835' = Trend View attributes

'836' = Premier attributes

'838' = Extended View attributes

'880' = Attributes

'881' = Model Attributes

'890' = STAGG Attributes

Value

Contains the computed result for the current attribute.

 

Typically, content length will be a minimum of nine characters and a maximum of fifteen, depending on which service generated the data. The lengths can vary further when a custom solution is being reflected, such as Instant Prescreen or Prequalification.
