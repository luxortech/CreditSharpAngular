Mop (Manner of Payment) Codes

 

    

 

MOP (Manner of Payment) codes are used to indicate the severity of any derogatory information for a given month.  These codes are used in ComTradeline instances within CurrentMop, MaximumDeliquencyMop and PaymentHistory elements as well as the Severity property of any ComDelinquency instances.

 

The table below illustrates the possible values that can be reported as MOP codes.  Not all values will be reported in each field.  If the MOP code has an X in the column corresponding to the fields mentioned, then it may be reported in that field.

 

Please note the slight but important difference between the zero based and one based values.  When items are examined directly through an instance of the CommonData or MergeData objects, the codes will be displayed using the Zero based definitions.

 

When items are displayed in any of the output formats (Json, Html, Text or Xml), the definition used for these values will be determined by the ZeroBasedMop property setting of the associated format option.  For backwards compatibility purposes, the CreditSharp framework defaults to using the one based definitions (hence ZeroBasedMop is defaulted to false).
