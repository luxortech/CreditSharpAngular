Attribute Factory

 

    

 

The CreditSharp Framework's Attribute Factory is a powerful and flexible utility for automating and standardizing the analysis of consumer credit information.  This feature exposes more than 1,800 credit data characteristics that can be used for modeling or as part of an intelligent decision making process.  

 

The variables are derived from the common data elements found within any credit bureau data response.  The definitions for each characteristic are evaluated against normalized elements ensuring consistent evaluation of a consumer's credit history, regardless of which credit bureau data source is being used.  All attributes can be generated using merged or unmerged data sets.

 

Some basic attributes:

TRL007

Total number of new tradelines opened in past 6 months

TRL008

Total number of new tradelines opened in past 12 months

TRL009

Total number of new tradelines opened in past 24 months

EDU405

Percent of education tradelines ever delinquent or defaulted

EDU406

Percent of education tradelines never worse than 30 days late

EDU407

Percent of education tradelines never worse than 60 days late

 

Attributes are organized by data category (Revolving, Mortgage, Bankruptcy, etc) and further grouped by the type of value being measured (Counts, Amounts, Delinquencies, etc).  There are 23 different data categories and 5 different value type measurements.  Attributes names are constructed by combining the two designations, where the data category is a three character alphabetic value following by a three digit numeric value.  The character portion indicates the data category and the numeric portion indicates the measurement type.  For example, the Attribute "REV304" provides an Amount (300+ range) value from the Revolving Account category.

 

An example of a typical programming pattern

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Analyze;

using LuxorTech.CreditSharp.Datamart.Common;

 

namespace HelpDemos

{

  public class PeformAnalysis

  {

      public AnalyzeData Execute(CommonData common)

      {

          AnalyzeOptions options = new AnalyzeOptions();

          options.IgnoreLostStolen = true;   // Do not count trades reported lost or stolen

 

          List<AnalysisBundle> bundles = new List<AnalysisBundle>();

          bundles.Add(new AnalysisBundle(AnalysisId.AttributeFactory01, Partitioning.IdMatch, options));

 

          AnalyzeData analyze = new AnalyzeData(bundles);

 

          analyze.Evaluate(common);

          return analyze;

      }

   }

}

 

After this logic has been executed, the calculated values could be viewed directly by inspecting each AnalyzeAttribute instance presented within the returned object, or, the returned object can be included as an parameter when generating FormatData outputs.
