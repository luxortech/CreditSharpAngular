LuxorTech.CreditSharp.Datamart.Merge Namespace

 

 

    

 

The Merge namespace is where any duplicate references that exist within a normalized common credit object can be eliminated.  Not all data elements are subject to de-duping; those that are not are simply copied in their entirety into the newly create merge object.  The MergeSection class can be used to selectively control merge behavior on a segment by segment basis.

 

Each Merge specific data class is derived from its Common counterpart and is then extended to include two additional properties in each segment: a BureauId property to identify if multiple credit bureaus provided information to a given item, and XtSources which maintains a record of each unmerged data record which was merged into the current item.

 

The Default Merge Operations topic provides a quick overview of what the CreditSharp framework uses for its default settings.
