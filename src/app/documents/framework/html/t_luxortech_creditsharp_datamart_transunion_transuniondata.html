TransUnionData Class

 

Namespace: LuxorTech.CreditSharp.Datamart.TransUnion

    

 

 

An instance of the BureauData class containing an TransUnion credit bureau image.

 

Constructor

 

Name

Description

TransUnionData(BureauOptions options)

Constructor for the TransUnionData class.

 

Syntax

public TransUnionData(BureauOptions options)

 

Parameters:

options

Type: Instance of the BureauOptions class

Container for applying options available when working with an TransUnion data set.

 

Example

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Datamart;

using LuxorTech.CreditSharp.Datamart.Equifax;

using LuxorTech.CreditSharp.Datamart.Experian;

using LuxorTech.CreditSharp.Datamart.TransUnion;

using LuxorTech.CreditSharp.Datamart.Common;

using LuxorTech.CreditSharp.Format;

 

namespace HelpDemos

{

  public class DemoCommon

  {

      public string GetXmlOutput(string rawCreditInput, BureauIds bureauId)

      {

          BureauData bureauData = null;

          switch (bureauId)

          {

              case BureauIds.Equifax:

                  bureauData = new EquifaxData(new BureauOptions(licenseKey));

                  break;

              case BureauIds.Experian:

                  bureauData = new ExperianData(new BureauOptions(licenseKey));

                  break;

              case BureauIds.TransUnion:

                  bureauData = new TransUnionData(new BureauOptions(licenseKey));

                  break;

          }

          bureauData.Parse(rawCreditInput);

 

          CommonData commonData = new CommonData(new CommonOptions());

          commonData.Normalize(bureauData);

 

          FormatOptions options = new FormatOptions(FormatLayout.Xml);

          options.Xml.Partitions = Partitioning.IdMatch;

          FormatData formatData = new FormatData(options);

 

          return formatData.Generate(commonData);

      }

   }

}

FilteredView

Bool

Gets or sets the property indicating if data elements will be presented in their cleansed or raw formats.

 

When true, elements will be displayed in consistent, type appropriate formats.  Amounts and numbers will be true numeric values with leading and trailing spaces trimmed and any leading zeroes removed. Character based fields will likewise have all leading and trailing spaces removed.  Date fields will conform to whatever DateMask option was chosen.

 

When false, all elements will be displayed in whatever raw form was provided by the bureau.

 

[Defaults to true]
