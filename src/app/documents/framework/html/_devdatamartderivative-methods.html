Derivative Methods

 

Namespace: LuxorTech.CreditSharp.Datamart

    

 

Derivative methods provide a means to obtain computed or conforming values from various elements within an instance of normalized data.  The values presented by Derivative methods can be used as the basis for creating unique analysis rules or aid in the construction of grouping and sorting criteria for customizing a formatted output.

 

Computed values return the result of a calculation, such as how long ago a given account was established (AGE).  Conforming values provide a numeric (integer) equivalent of the underlying data element.

 

By default, all bureau data elements within the CreditSharp framework are presented as character strings, regardless of the actual content of a given field.

 

All dates, amounts and numbers are stored internally in type appropriate containers.  Externally, the are displayed as strings because it is often necessary to distinguish between instances where a bureau has failed to provide an actual value for a given data point.  For example, it can be informative to known when a tradeline balance has explicitly reported as zero (0) as opposed to not being reported at all.  If that data point was always defined as an integer value, it becomes difficult to know when zero truly means "zero" or when it means "not reported".

 

Method Name

Description

AGE

Computes the number of months between a given date field and the original order date of the credit report.

AGE2

Computes the number of days between a given date field and the original order date of the credit report.

AMT

Returns the integer equivalent of an underlying dollar amount element.

DATE

Returns an integer equivalent of an underlying date element.

 

The value is presented as the number of months elapsed since January 1900.

DATE2

Returns an integer equivalent of an underlying date element.

 

The value is comprised of the CCYYMMDD digits converted to a number, e.g. 20,170,819.

EMPTY

Returns a boolean value indicating if a given element contains valid data or is blank.

INV

Returns a logically inverted representation of a character string.

 

This is used when sorting data in reverse alphabetical order.

NBR

Returns the integer equivalent of an underlying numeric element.
