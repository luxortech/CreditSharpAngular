LuxorTech.CreditSharp.Analyze Namespace

 

    

 

The Analyze namespace is used when business requirements dictate the need for aggregated credit data variables or when summary data is required for inclusion in the various output layouts. There are two predefined sets of credit attributes which can be executed, or, custom aggregate definitions can be defined and evaluated.

 

•AttributeFactory01: A rich collection of over 1,800 predefined attributes suitable for modeling or deeper analysis.

•SummaryData01: Provides summarized results for populating the summary sections of CreditSharp outputs.

 

An example of a typical programming pattern

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Analyze;

using LuxorTech.CreditSharp.Datamart.Common;

 

namespace HelpDemos

{

  public class PeformAnalysis

  {

      public AnalyzeData Execute(CommonData common)

      {

          AnalyzeOptions options = new AnalyzeOptions();

          options.IgnoreLostStolen = true;   // Do not count trades reported lost or stolen

 

          List<AnalysisBundle> bundles = new List<AnalysisBundle>();

          bundles.Add(new AnalysisBundle(AnalysisId.SummaryData01, Partitioning.IdMatch, options));

 

          AnalyzeData analyze = new AnalyzeData(bundles);

 

          analyze.Evaluate(common);

          return analyze;

      }

  }

}

 

After this logic has been executed, the calculated values could be viewed directly by inspecting each AnalyzeAttribute instance presented within the returned object, or, the returned object can be included as a parameter when generating FormatData outputs.

 

Analyze Classes

 

 

Class

Description

AnalysisBundle

Container class for selecting and configuring a set of attributes to be evaluated.

AnalyzeData

Root class used to perform selected calculations on a normalized credit bureau data image.

AnalyzeOptions

Container class for specifying various options for controlling Analyze features and functions.

AnalyzeResult

Container class for reviewing calculation results. A reference to this object can be passed to the FormatData type so that the values can be included in a predefined output.

AnalyzeAttribute

Class used to present the outcome of an individual calculation.

The Evaluate classes are only referenced if constructing custom Analysis bundles.

EvaluateIdentification

Class used to create and define a rule to be used for analyzing ComIdentification segment content.

EvaluateInquiry

Class used to create and define a rule to be used for analyzing ComInquiry segment content.

EvaluatePublicRecord

Class used to create and define a rule to be used for analyzing ComPublicRecord segment content.

EvaluateReference

Class used to create and define a analysis rule which references results of previously calculated rules.

EvaluateTradeline

Class used to create and define a rule to be used for analyzing ComTradeline segment content.

 

Analyze Delegates

 

Delegates are used to create custom definitions of credit data aggregates.  The predicates define which occurrences of a given credit segment will be measured and solutions define the computation that will be performed on any selected occurrences.

 

Delegate

Description

ComputePredicate

Accepts a reference to a lambda expression or compiled method to be used in determining if the current AnalyzeResult instance should be evaluated.

ComputeSolution

Accepts a reference to a lambda expression or compiled method which will compute and return a value using the selected AnalyzeResult instance.

IdentificationPredicate

Accepts a reference to a lambda expression or compiled method to be used in determining if the current ComIdentification instance should be evaluated.

IdentificationSolution

Accepts a reference to a lambda expression or compiled method which will compute and return a value using the selected ComIdentification instance.

InquiryPredicate

Accepts a reference to a lambda expression or compiled method to be used in determining if the current ComInquiry instance should be evaluated.

InquirySolution

Accepts a reference to a lambda expression or compiled method which will compute and return a value using the selected ComInquiry instance.

PublicRecordPredicate

Accepts a reference to a lambda expression or compiled method to be used in determining if the current ComPublicRecord instance should be evaluated.

PublicRecordSolution

Accepts a reference to a lambda expression or compiled method which will compute and return a value using the selected ComPublicRecord instance.

TradelinePredicate

Accepts a reference to a lambda expression or compiled method to be used in determining if the current ComTradeline instance should be evaluated.

TradelineSolution

Accepts a reference to a lambda expression or compiled method which will compute and return a value using the selected tradeline instance.

 

Analyze Enumerations

 

 

Enumeration

Description

AnalysisActive

Enumerations available for use when invoking the tradeline IsActive method, which indicate if any specific date(s) should be ignored.

AnalysisCreditCapacity

Specifies how available limit calculation is performed.

AnalysisDate

Used to specify which date will be the basis for age related calculations.

AnalysisDateUnit

Specifies what unit is being measured when comparing the gap between two dates

AnalysisId

AnalysisLateSeverity

Specifies what level of delinquent occurrence is being evaluated

AnalysisLimitProxy

Specifies if an alternate CreditLimit source can be used in relevant calculations.

AnalysisOperation

Indicates what type of instruction will be performed in a given evaluation.

AnalysisResultType

indicates the fundamental data type of an evaluation result.

AnalysisTradeState

Specifies how the Open or Closed status of a given Tradeline is decided.

 

Analyze Operations

 

For a complete overview of custom operations, you can visit the Operations chapter.

 

Operation

Description

Average

Returns the average (mean) of all values in the indicated numeric element.

Compute

Returns the result of an equation.

Count

Returns number of items that match a given data condition.

Highest or Lowest

Returns the highest (or lowest) numeric value found within the indicated element.

Maximum or Minimum

The same as highest/lowest but ignores the AssumeZero and always skips over blank/missing elements.  (See Working with Blank or Missing data below.)

Newest or Oldest

Returns the most recent (or most ancient) date value found within the indicated element.

Sum

Returns the total of all values for the indicated numeric element.

Worst

Returns the most severe manner of payment.
