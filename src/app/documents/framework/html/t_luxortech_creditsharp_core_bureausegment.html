BureauSegment

 

    

 

The BureauSegment class is the abstract base class from which all occurrences of bureau specific segments are derived.  It used within the CreditSharp Framework as a means to bundle different segments of a bureau's response into a single container to make some type of analysis and retrieval easier.

 

The prime example of this is the Segments property found in each bureau's data object (EquifaxData, ExperianData and TransUnionData).  This property presents a collection of all the segments parsed from a given bureau response, that are displayed using the original order in which they appeared in the data.  If there is ever a reason to examine the bureau response in its original sequence (rather than inspecting elements grouped by type), this base class supports that goal.

 

For example:

 

Here we import an Equifax bureau image and then loop through the generic collection of parsed segments.  The complete content of each raw segment extracted is added to a growing output with each delimited by a new line character.  Along the way, we test to see if a segment is a Tradeline and if so, accumulate its balance into a total.

 

 

      public string GetDelimitedEfx(string creditBuffer)

      {

          EquifaxData efxData = new EquifaxData(new BureauOptions(licenseKey));

          efxData.Parse(creditBuffer);

 

          StringBuilder allSegments = new StringBuilder();

          int balances = 0;

 

          foreach (BureauSegment segment in efxData.Segments)

          {

              allSegments.AppendLine(segment.MtBureauSource + Environment.NewLine);

 

              if (segment is EfxTradeline)

              {

                  EfxTradeline tradeline = (EfxTradeline)segment;

                  int temp;

                  if (Int32.TryParse(tradeline.Balance, out temp) == true)

                      balances += temp;

              }

          }

          return allSegments.ToString();

       }

 

The properties below outline the members that can be accessed through this generic base class.  As there is no public constructor for this class, it can only be used as a target for upcasting and streamlining some processes, as demonstrated above.

DtSegmentId

Num (2)

Gets the value used to uniquely identify a given bureau segment instance.

 

The value expressed here is taken from the segment id enumerations specific to each bureau.  If reviewing an Equifax input, they will be one of the EfxSegmentIds enumerations.  Experian's will be from ExpSegmentIds and TransUnion's would be from TucSegmentIds.

DtSequence

Num (3)

Gets the value indicating the sequence number of the current segment relative to others of the same type.

 

As an example, if there are 14 tradeline occurrences in the current data set and the fifth iteration of these is currently being examined, this property would contain the value of 5.
