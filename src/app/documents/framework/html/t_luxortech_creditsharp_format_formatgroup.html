FormatGroup Class

 

Namespace: LuxorTech.CreditSharp.Format

    

 

The references used to define groups for certain sections within Html or Text layouts are all derived from the FormatGroup class.  The base class exposes the name and any headers for the group  while the derived class is used to define the type specific selection criteria for choosing members to include in the group.  Coding examples specific to each section can be found within the FormatSecton derived class which corresponds to the type of data being manipulated.

 

The following topics discuss the settings required to implement customized group criteria.  There are also a number of predefined criteria for grouping certain report sections. These existing criteria can be reviewed in the section specific topic pages found within the Derived Classes chapter - e.g. - to view possible existing groups that can be used with Tradelines, review the FormatSectionTradeline topic.

Constructing headers

The appropriate content for a group header is dictated, somewhat, by the layout in which it will be used.  Group headers in Text layouts can consist of multiple lines but are shown in the same fixed-font used for all other data being displayed.  On the other hand, headers used in Html layouts can only be a single line but can include any valid html tag(s), meaning the color and font can be customized as well as verbiage.

 

Headers for existing predefined groups can be modified, but the Selector cannot.  To disable a header, the value can be assigned as string.emtpy (in order to, for example, prevent a Missing header from being displayed).

 

An example of the more common form of group header used within CreditSharp Text layouts might be:

 

-------------

Open Accounts

-------------

 

Which would be assigned as:

 

group.InitialHeader = "-------------" + Environment.NewLine +

"Open Accounts:" + Environment.NewLine +

"--------------" + Environment.NewLine;

 

 

 

For Html layouts, headers will automatically be decorated with a line above and below, such as:

 

group.InitialHeader = "Open Accounts"

 

displays:

 

Open Accounts

 

 

Optionally, font size or color can be altered:

 

group.MissingHeader = "<span style=\"font-size: 12px; background: #c3d69b;\">MORTGAGE ACCOUNTS</span>";

 

displays:

 

Mortgage Accounts

 

 

Or, color can be applied selectively:

 

group.MissingHeader = "MORTGAGE ACCOUNTS: <span style=\"color:red\">None found</span>";

 

displays:

 

Mortgage Accounts: None found

 

 

Extended Property

Each derived FormatGroup class exposes its own Selector property, which accepts a delegate that provides the criteria for choosing the members of the group.

 

 

Name

Content

Description

Selector

Func

Defines criteria for choosing what members will be included in a given group.

 

The property values is expressed as Func <T, bool> where T is a reference to one of the common CreditSharp data instances.

 

Assigning the value of a group's Selector can be done using either lambda expressions or a function (or method) reference which matches the delegate signature.  Note that selectors cannot be modified in any predefined groups - a Custom group definition be created instead.

 

Generally, using references to functions or methods will lead to faster execution times as they are compiled during development rather than at run time.  Lambda expressions on the other hand must be compiled and resolved at run time.  However, the performance gain is likely to be very small since there will typically only be a few instances of these types of instructions.

 

In the following examples, we are establishing a group which isolates unpaid collection accounts.

 

Selector using LINQ

Lambda syntax 1

 

tradeGroup.Selector = (x => x.IsCollection && x.AMT(ComTradelineIds.BalanceAmount) > 0);

 

Lambda syntax 2

This style is more verbose, but does make it very clear which CreditSharp object is being evaluated.

 

tradeGroup.Selector = (ComTradeline x) => { return x.IsCollection && x.AMT(ComTradelineIds.BalanceAmount) > 0); };

 

Selector using Delegate

 

The first step in using a delegate for group selection, is to create the function which tests for the desired criteria.  Then, this function can be assigned to the Selector property.

 

private bool IsOpenCollection (ComTradeline trade)

{

    if (trade.IsCollection && trade.AMT(ComTradelineIds.BalanceAmount) > 0)

        return true;

    else

        return false;

}

 

 

Delegate syntax 1

 

tradeGroup.Selector = IsOpenCollection;

 

Delegate syntax 2

This style is more verbose, but helps to more explicitly state what is being executed.

 

Func<ComTradeline, bool> mySelector = IsOpenCollection;

tradeGroup.Selector = mySelector;
