Format Enumerations

 

Namespace: LuxorTech.CreditSharp.Format

    

 

The following tables list all the enumerated values available for use when generating output with a FormatData object.

 

Enumeration

Description

FormatLayout

Indicates which output format is desired.

Member

Value

Description

Xml

0

Specifies that the output be constructed as XML.

Text

1

Specifies that the output be a text based image, suitable for printing or GUI displays.

Html

2

Specifies that the output be an HTML image.

Json

3

Specifies that the output be in JavaScript Object Notation form.

Dataset

4

Specifies that the output be a .NET DataSet (System.Data.DataSet).

HtmlScoreRankDisplay

Indicates if a separate status element should be displayed in the tradeline section of an Html layout.

Member

Value

Description

Standard

0

Do not return any consumer referral contact information. (Default)

Shaded

1

The percentile field will be given a uniform color background to highlight into what range the value falls.  When red, the percentile falls into the lowest (poorest) range, green for the highest (best) range, and yellow for everything between.

Gradient

2

The percentile field will be shown with a color background which progresses from red on the left to green on the right, and the data value positioned proportionally within the gradient (left side indicating lower rankings and right side being higher).

HtmlStyle

Indicates which report layout will be used when generating the Html output.

 

Member

Value

Description

Common01

0

Specifies the default CreditSharp layout, which displays Primary and Secondary data separately, effectively generating two reports.

Lending01

1

Mimics to the basic Common01 layout but with Primary and Secondary details listed together in each data section.

 

HtmlTradeDimensions

Indicates if and how the Historical Activity display should be included in the tradeline detail of an Html layout.

Member

Value

Description

None

0

No dimension display is included, even if underlying data is present. (Default)

Expanded

1

Includes the dimension display in expanded form, when underlying data is present.

Collapsed

2

Includes the dimension display in collapsed form, requiring a user click to reveal content, when underlying data is present.

HtmlTradeStatusSource

Indicates how a tradeline status display is chosen in Html layouts.

Member

Value

Description

CurrentStatus

0

Submit the request to the test system (Default).

WorstStatus

1

Submit the request to the production system.

When the HtmlTradeStatusDisplay is enabled, the code can be determine the status display by inspecting just the current status or both the current and former (maximum) status.

HtmlTradeStatusDisplay

Indicates if a separate status element should be displayed in the tradeline section of an Html layout.

Member

Value

Description

None

0

Do not return any consumer referral contact information. (Default)

Graphic

1

Return the referring bureau's phone and address detail.

Text

2

Return the referring bureau's website along with the phone and address.

HtmlTradelineStyle

Specifies which tradeline layout will be employed in Html outputs.

Member

Value

Description

Tradeline01

0

Selects the tradeline layout that shows a single line of detail in the collapsed / summary view.

Tradeline02

1

Selects the tradeline layout showing two lines of detail in the collapsed view, with Creditor Name on the first line.

Tradeline03

2

Selects the tradeline layout that shows two lines of detail in the collapsed view, with Creditor Name and Account Number on the first line.

Tradeline04

3

Selects the tradeline layout fashioned after Tradeline02, with Balance added to the second line.

Tradeline05

4

Selects the tradeline layout fashioned after Tradeline03, with Balance added to the second line.

Tradeline06

5

Selects the tradeline layout fashioned after Tradeline05, with Payment amount and Payment history included in the summary lines.


JsonMergeHistory

Specifies if any merge history should be included in the JSON output.

Member

Value

Description

None

0

No merge history will be generated. (Default)

Stack

1

Specifies that the single deduped entities from each bureau be displayed.

Origin

2

Specifies that all unmerged antecedents be included in the output.

StackAndOrigin

3

Specifies that both the Stack and Origin entities be included in the output.

When merge history is displayed, one or more child nodes are included beneath a given output segment showing which individual segments were combined or merged to create the current parent.

JsonStyle

Used to indicate what JSON layout is being requested.

Member

Value

Description

0

Member

Value

Description

0

Lending01

1

Specifies a layout patterned after Common01, with the addition of a separate Identification section.
