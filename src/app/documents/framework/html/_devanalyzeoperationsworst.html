Worst

 

Namespace: LuxorTech.CreditSharp.Analyze

    

 

The Worst operation will return the most severe delinquent occurrence found in a tradeline manner of payment value.  These values can be found in the CurrentMop, MaximumDelinquencyMop and PaymentPattern data elements.  The computation is defined using the AddEvaluation method within the AnalysisBundle class.

 

In order to account for all possible data conditions in the tradeline manner of payment elements, the NBR() helper function should be used in conjunction with the Worst operation.  This will ensure that alphabetic mop codes are properly handled.  In addition, the AssumeZero option property found on the CommonData or MergeData container classes should be set to true so that any alphabetic values are treated as zero.

 

List of MOP codes in severity order (worst to least): 7 > 8 > 9 > 6 > 5 > 4 > 3 > 2 > 1 > 0

 

As shown above, a Bankruptcy (7) is the most severe manner of payment code that can be found.  After that, Repossession or Foreclosure (8) is next most severe.  Then Charge-off or Collection (9).  And finally number of days late, in reverse order.  So, 180 days is worse than 150 days is worse than 120 days and so on.

 

 

To get the worst current delinquency of any revolving tradeline:

 

TradelinePredicate isRevolving = (trade, options) => trade.IsRevolving;

bundle.AddEvaluation(new EvaluateTradeline("WorstLate", AnalysisOperation.Worst, ComTradelineIds.CurrentMop, isRevolving));
