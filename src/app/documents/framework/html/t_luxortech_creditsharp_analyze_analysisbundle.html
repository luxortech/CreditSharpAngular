AnalysisBundle Class

 

Namespace: LuxorTech.CreditSharp.Analyze

    

An AnalysisBundle serves as container for a group of evaluations that should be performed.  This can be one of the predefined CreditSharp evaluation groups (Attribute Factory or Summary Data) or a entirely custom set of evaluation criteria, as indicated with the AnalysisId parameter.  By specifying a Partitioning scheme, evaluations for a given bundle can be restricted to  individual applicants or the credit data content in its entirety.

Example

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Analyze;

using LuxorTech.CreditSharp.Datamart.Common;

 

namespace HelpDemos

{

  public class PeformAnalysis

  {

      public AnalyzeData Execute(CommonData common)

      {

          AnalyzeOptions options = new AnalyzeOptions();

          options.IgnoreLostStolen = true;   // Do not consider trades reported lost or stolen

          options.IgnoreDisputed = true;     // Do not consider trades in dispute

 

          List<AnalysisBundle> bundles = new List<AnalysisBundle>();

          bundles.Add(new AnalysisBundle(AnalysisId.SummaryData01, Partitioning.IdMatch, options));

          bundles.Add(new AnalysisBundle(AnalysisId.AttibuteFactory01, Partitioning.Report, options));

 

          AnalyzeData analyze = new AnalyzeData(bundles);

 

          analyze.Evaluate(common);

          return analyze;

      }

  }

}

 

Remarks

Ultimately, the final content that appears in the Result property of the AnalyzeData object after the Evaluate method has completed its work is determined by two values: the Partitioning parameter provided to the AnalysisBundle constructor and the IsolateJoint property within the AnalyzeOptions class.  

 

When evaluating a merged data set, the IsolateJoint property controls how observations that are shared between the primary and secondary applicant are to be considered.  If the IsolateJoint property is true, then all observations within the credit data that are jointly held between the primary and secondary applicants will be evaluated separately and all resulting attributes will be returned in their own AnalyzeResult instance in the AnalyzeData.Results list.

 

Similarly, the Partitioning selection also affects how many AnalyzeResult instances will appear in the Results list.  When the value is Report, there will only be a single AnalyzeResult, since the Report option means that all data will evaluated at once (this setting also supersedes the IsolateJoint value). On the other hand, when the value is IdMatch there will be one AnalyzeResult instance for each applicant.
