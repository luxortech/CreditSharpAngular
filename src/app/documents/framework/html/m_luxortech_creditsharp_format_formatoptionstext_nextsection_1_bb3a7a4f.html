FormatOptionsText.NextSection Method

 

Namespace: LuxorTech.CreditSharp.Format

    

 

The NextSection method is invoked when adding additional display sections to a customized output order.  After calling the FirstSection method to initialize the display sequence, additional calls are made to the NextSection method to add one or more displays sections in the desired order.

 

If the goal is only to turn off display of particular items rather than modifying the complete sequence, the Section method can be used to selectively disable individual display sections.

 

The order of sections in the Common01 output variants is shown below, as well as the section order for Lending01 variants.

 

Common01 Section order

Description

Displays any error information reported by a credit bureau.

Identification

Displays the consumer name, age, and social security numbers.

Address

Displays the consumer's current and former residential address history.

Alert

Displays outcome of any fraud search services, active duty notices, etc.

Summary

Displays a table with summary level elements.

ConsumerStatement

Displays any consumer statement information included in the bureau data.

Score

Displays the results of all bureau score model requests.

Employment

Displays the consumer's employment history when available.

PublicRecord

Displays any legal actions record against the consumer.

Tradeline

Displays details of all credit account and payment history.

Banking

Displays details of banking (checking and savings) history.  (Only appears in Canadian credit data.)

RegisteredItem

Displays details of secured loans (chattel) that have been listed in the public registry.  (Only appears in Canadian credit data.)

Message

Displays general items that don't fit within other report categories.

Inquiry

Displays a record of all what other entities or business have checked the consumer's credit. history.

OfacSearch

Displays the outcome of an Office of Foreign Assets Control search.

ConsumerReferral

Displays the information for a consumer to use when contacting the bureau furnishing the data.

Directory

Displays the full name, address and phone number (as available) for all entities that reported information included in the output.

If so desired, this order can be modified by using the FirstSection and NextSection methods. Calling FirstSection effectively turns off the display of all default sections and then establishes the one indicated as the first display item.  Subsequent calls to the NextSection method would add additional sections to be displayed.

 

Syntax

public void FirstSection (string sectionName)

 

Parameters:

 

sectionName

Type: string

Identifies the report section that should be displayed next.  Must be one of the known sections (refer to either the Common01 or Lending01 section tables above).

Example

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Datamart.Common;

using LuxorTech.CreditSharp.Format;

 

namespace HelpDemos

{

  public class DemoFormat

  {

      public string CreateText(CommonData common)

      {

          FormatOptions options = new FormatOptions(FormatLayout.Text);

          options.Text.Partitions = Partitioning.IdMatch;

          options.Text.ReportStyle = TextStyle.Lending01;

 

          this.SetCustomText(options);

 

          FormatData format = new FormatData(options);

          return format.Generate(common);

      }

 

      public void SetCustomText(FormatOptions options)

      {

          // Only display four sections

          options.Text.FirstSection("Header");

          options.Text.NextSection("Score");

          options.Text.NextSection("Summary");

          options.Text.NextSection("Tradelines");

      }

  }

}
