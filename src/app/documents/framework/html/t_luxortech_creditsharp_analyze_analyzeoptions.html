AnalyzeOptions Class

 

Namespace: LuxorTech.CreditSharp.Analyze

    

 

The AnalyzeOptions type exposes the following members used to control various aspects of the Analyze component's evaluations.  An instance of this class must be populated for use when creating an AnalysisBundle object.

 

If you will be evaluating more than one bundle, the same AnalyzeOptions instance can be passed into each bundle, or, separating option instances can be created so that each bundle can be evaluated using different options.

Example

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Analyze;

using LuxorTech.CreditSharp.Datamart.Common;

 

namespace HelpDemos

{

  public class PeformAnalysis

  {

      public AnalyzeData Execute(CommonData common)

      {

          List<AnalysisBundle> bundles = new List<AnalysisBundle>();

 

          AnalyzeOptions summaryOptions = new AnalyzeOptions();

          summaryOptions.IgnoreLostStolen = true;   // Do not consider trades reported lost or stolen

          summaryOptions.IgnoreDisputed = true;     // Do not consider trades in dispute

 

          bundles.Add(new AnalysisBundle(AnalysisId.SummaryData01, Partitioning.IdMatch, summaryOptions));

 

          // Create different set of options for use with the second bundle

          AnalyzeOptions factoryOptions = new AnalyzeOptions();

          factoryOptions.IgnoreLostStolen = false;   // Do not consider trades reported lost or stolen

          factoryOptions.IsolateJoint = true;         // Do not consider trades in dispute

          bundles.Add(new AnalysisBundle(AnalysisId.AttibuteFactory01, Partitioning.IdMatch, factoryOptions));

 

          AnalyzeData analyze = new AnalyzeData(bundles);

 

          analyze.Evaluate(common);

          return analyze;

      }

   }

}

BaseDate

Enum

One of the AnalysisDate enumerations.

Indicates what date will be used to determine the relative age of various credit report elements.

 

[Defaults to AnalysisDate.Report]

DefaultEmptyCalcs

Bool

Gets or sets the value indicating if numeric calculations should default to zero when no observations exist within the credit data.

 

[Defaults to true]

 

Note: An example of this default usage might be when attempting to, say, add up the total balances on auto loan accounts and there are no auto loans present in the bureau data.  If this property set to true then outcome would be reported as zero and when set to false the outcome would be left blank.

DelinquencyMonths

Num (2)

For Summary evaluations, indicates how much account history should be included in calculations.  

 

[Defaults to 24 months]
