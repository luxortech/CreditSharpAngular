MergeOptions Class

 

Namespace: LuxorTech.CreditSharp.Datamart.Merge

    

 

The MergeOptions class provides access to setting that can be used to control features within the Merge namespace.

CombineProfile

Text

Offers a short hand way of assigning the Combine settings for all sections with a single instruction.

 

Valid inputs:

Pick

Applies the CombineMethod.Pick enumeration to all sections.

Blend

Applies the CombineMethod.Blend enumeration to all sections.

System

Restores/applies the default settings to all sections.

 

The default values can be reviewed in the Default Merge Operations topic.

 

The value can be used to modify how duplicate items are resolved once identified, either using a Pick or Blend methodology (as described in the CombineMethod topic).  Note that use of this property will only modify the sections where the new value is appropriate.  For example, Blend is not a valid option for Inter-bureau Inquiry segment resolutions, so that section's value would not be changed.  (A list of valid settings is in the MergeSecton topic.)

 

If desired, a different methodology can be used for intra (within bureau) and inter (across bureau) operations.  To specify different criteria, the property can be provided two values separated by the plus character, as in:

 

MergeOptions.CombineProfile = "Blend+Pick";

 

This would indicate the all intra-bureau pairs be resolved using the Blend option and all inter-bureau (between bureau) pairs be resolved using the Pick method.

 

To assign both intra and inter bureau operations the same method, then only one input is needed:

 

MergeOptions.CombineProfile = "Blend";

 

[Defaults to "System"]

CompareProfile

Text

Offers a short hand way of assigning the Compare settings for all sections with a single instruction.

 

Valid inputs:

Basic

Applies the CompareMethod.Basic choice in all sections.

Standard

Applies the CompareMethod.Standard choice in all available sections.

 

Some sections cannot be evaluation using the Standard comparison level. Instead those sections adopt the next highest possible level (Basic).

Advanced

Applies the CompareMethod.Advanced choice in all available sections.

 

Some sections cannot be evaluation using the Advanced comparison level. Instead those sections adopt the next highest possible level (Standard or Basic).

System

Restores/applies the default settings to all sections.

 

The default values can be reviewed in the Default Merge Operations topic.

 

The value can be used to modify the method employed in identifying duplicate items, as described in the CompareMethod topic.  (For reference, a list of valid settings for each section is in the MergeSecton topic.)

 

If desired, a different methodology can be used for intra (within bureau) and inter (across bureau) operations.  To specify different criteria, the property can be provided two values separated by the plus character, as in:

 

MergeOptions.CompareProfile = "Standard+Advanced";

 

This would indicate the all intra-bureau duplicates be identified using the Standard merge criteria and all inter-bureau (between bureau) duplicates be identified using the Advanced criteria.

 

To assign both intra and inter bureau operations the same criteria, then only one input is needed:

 

MergeOptions.CompareProfile = "Basic";

ZeroBasedMop

Bool

Get or sets the value which determines if a Tradeline Manner of Payment and Payment History codes will be numbered from zero or one.

 

When true, an instance of a current month would be represented as "0" while a 30 day late would be seen as "1", a 60 day late as "2", etc.  When false, the current month would be styled as "1" with 30 day lates being seen as "2" and 60 day lates being "3", etc.

 

Having the codes zero based allows for the explicit representation of a 180 day delinquent occurrence, as "6".  When the codes are based from one, then the value of "6" must stand in for either 150 or 180 days late.  

 

Please refer to the Mop Codes appendix topic for a full presentation of this optional feature.

 

[Defaults to false]
