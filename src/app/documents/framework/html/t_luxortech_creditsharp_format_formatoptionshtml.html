FormatOptionsHtml Class

 

Namespace: LuxorTech.CreditSharp.Format

    

 

The members found within the FormatOptionsHtml class provide access to settings for controlling the creation of Html based outputs.

ScoreRankDisplay

Enum

One of the HtmlScoreRankDisplay enumerations.

Gets or sets the value indicating how the score rank (percentile) will be shown.

 

[Defaults to Shaded]

ScoreRankLower

Int (2)

Gets or sets the value specifying the lower threshold for percentiles to be considered 'average'.

 

When the ScoreRankDisplay enumeration is Shaded, any score percentiles that are lower than this value will be displayed with the red tinted background, to indicate the value falls into the poorest possible score ranking range.

 

Values between the ScoreRankLower and ScoreRakUpper (inclusive) will be shaded with the yellow background.

ScoreRankUpper

Int (2)

Gets or sets the value specifying the upper threshold for percentiles to be considered 'average'.

 

When the ScoreRankDisplay enumeration is Shaded, any score percentiles that are higher than this value will be displayed with the green tinted background, to indicate the value falls into the best possible score ranking range.

 

Values between the ScoreRankLower and ScoreRakUpper (inclusive) will be shaded with the yellow background.

ScriptLocation

Text

Gets or sets the value indicating where the Javascript code for supporting dynamic Html is stored.

 

[Defaults to "inline"]

 

When left as 'inline', the necessary scripting entries will be embedded within the generated Html output.  Alternately, the property can be assigned as a path and file name which then is inserted as a reference within the Html.  Using an external script file can save storage space by not having a full copy of the script in every formatted credit report.

 

The external script file is included along with the basic CreditSharp installation, in the Assets folder:

C:\Program Files (x86)\Luxor Technologies, Inc\Assets

 

The file can be placed anywhere within the active file system or referred to from the installation folder, such as:

 

FormatOptions options = new FormatOptions(FormatLayout.Html);

options.Html.ScriptLocation = @"C:\Program Files (x86)\Luxor Technologies, Inc\CreditSharp\Assets\CSHtmlScript.js";

 

The file can also be part of a content delivery network, and specified as a URL, such as:

 

FormatOptions options = new FormatOptions(FormatLayout.Html);

options.Html.ScriptLocation = "http://www.example.com/CSHtmlScript.js";

 

In both instances, providing a relative path or url is acceptable.

StyleLocation

Text

Gets or sets the value indicating is the CSS Stylesheet for rendering an Html output is stored.

 

[Defaults to "inline"]

 

When left as 'inline', the necessary stylesheet entries will be embedded within the generated Html output.  Alternately, the property can be assigned as a path and file name which then is inserted as a reference within the output.  Using an external stylesheet can save storage space by not having a full copy in every formatted credit report.

 

The external stylesheet is included along with the basic CreditSharp installation.  The file can actually be placed anywhere within the active file system, or referred to from it's installed location, such as:

 

FormatOptions options = new FormatOptions(FormatLayout.Html);

options.Html.StyletLocation = @"C:\Program Files (x86)\Luxor Technologies, Inc\CreditSharp\Assets\CSHtmlStyle01.css";

 

The file can also be part of a content delivery network, and specified as a URL, such as:

 

FormatOptions options = new FormatOptions(FormatLayout.Html);

options.Html.StyleLocation = "http://www.example.com/CSHtmlStyle02.css";

 

In both instances, provided a relative path or url is acceptable.

 

Note: There are two style sheets (CSHtmlStyle01.css and CSHtmlStyle02.css).  The '01' style is to be used when generating HtmlTradelineStyle.Tradeline01 outputs.  The '02' style is for all other HtmlTradelineStyle options.  If left as inline, the software will automatically pick up the correct style sheet.

TradelineStatusField

Enum

One of the HtmlTradeStatusSource enumerations.

Gets or sets the values which control the tradeline status is determined.

 

[Defaults to WorstStatus]

TradelineStatusMajor

Int (1)

Gets or sets the value which determines if an account should be considered a minor derogatory item or a major derogatory item.

 

This setting corresponds to the tradeline Current or Maximum Mop value.  Accounts with Mop codes equal to or greater than the indicated value will be considered Major derogatory items.

 

Note that the runtime evaluation of this value is also dependent upon the ZeroBasedMop property.  When ZeroBasedMop is true, the Status threshold chosen here will also be compared from a zero base (where 0 is current, 1 is 30 days late, 2 is 60 days late, etc).

 

[Defaults to 6] (assumes the ZeroBasedMop setting is false)

TradelineStatusMinor

Int (1)

Gets or sets the value which determines if an account should be considered a minor derogatory item or a major derogatory item.

 

This setting corresponds to the tradeline Current or Maximum Mop value.  Accounts with Mop codes equal to or greater than the indicated value (and less than TradelineStatusMajor) will be considered Minor derogatory items.

 

Note that the runtime evaluation of this value is also dependent upon the ZeroBasedMop property.  When ZeroBasedMop is true, the Status threshold chosen here will also be compared from a zero base (where 0 is current, 1 is 30 days late, 2 is 60 days late, etc).

 

[Defaults to 2] (assumes the ZeroBasedMop setting is false, so two here means 30 days late.)

ZeroBasedMop

Bool

Get or sets the value which determines if a Tradeline Manner of Payment and Payment History codes will be numbered from zero or one.

 

When true, an instance of a current month would be represented as "0" while a 30 day late would be seen as "1", a 60 day late as "2", etc.  When false, the current month would be styled as "1" with 30 day lates being seen as "2" and 60 day lates being "3", etc.

 

Having the codes zero based allows for the explicit representation of a 180 day delinquent occurrence, as "6".  When the codes are based from one, then the value of "6" must stand in for either 150 or 180 days late.  This does also affect the TradelineStatusMajor and TradelineStatusMinor options, as whatever desired cutoffs are chosen will be compared to either the zero of one based statuses.

 

Please refer to the Mop Codes appendix topic for a full presentation of this optional feature.

 

[Defaults to false]
