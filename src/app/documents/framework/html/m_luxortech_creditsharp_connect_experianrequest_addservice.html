AddService Method

 

Namespace: LuxorTech.CreditSharp.Connect

    

 

The AddService method is used to include a specific service request within the Experian inquiry that represents a unique standalone product or is an Experian feature which requires additional inputs not already accounted for in the ExpInquiryOptions class.  The method returns an instance of a BureauService object for the newly created service request.  With this reference, other properties of the service (such as custom inputs) can be assigned.

 

If the feature being requested is a standalone product then the ExperianRequest.ProductCode should be assigned as ExpProductCode.Service.  If, on the other hand, the feature being requested in simply an add-on to the standard credit report, the product code should be left as ExpProductCode.CreditReport.  In most cases, the service request specified here will be for a standalone product so the Service enumeration would likely be appropriate.

 

 

Syntax

public BureauService AddService (String serviceName, String serviceLabel, String serviceId)

 

Parameters:

 

serviceName

Type: string

Contains the name of the Experian feature being requested.

 

serviceLabel

Type: string

Contains the label identifying the Experian feature being requested.  This value would correspond to either the NetConnect CustomRRKeyword or the APIHUB solutionId.

 

serviceId

Type: string [Optional]

Contains the unique customer specific id for the feature being requested.  This value would correspond to the NetConnect SolutionId or the APIHUB daasReferenceId.

Example

 

  public class ExperianService

   {

      public string GetCustomData(string licenseKey, Applicant applicant)

       {

          BureauConnection connection = new BureauConnection(BureauIds.Experian);

           connection.HttpSettings.AuthorizationId = "myAuthId";

           connection.HttpSettings.AuthorizationSecret = "myAuthSecret";

           connection.HttpSettings.ServiceName = "API";

           connection.HttpSettings.ServiceURL = "https://stg-us-api.experian.com/oauth2/v1/token";

           connection.HttpSettings.WebUserId = "myCompanyId";

           connection.HttpSettings.WebPassword = "notPassword";

 

          ExpInquiryOptions options = new ExpInquiryOptions(licenseKey);

           options.ServiceRegion = ExpServiceRegion.Production;

 

          ExperianRequest request = new ExperianRequest(options);

           request.PermissiblePurposeCode = "98";

           request.ProductCode = ExpProductCode.CreditReport;

           request.JsonResponse = ExpJsonResponse.Standard;

           request.ResponseVersion = ExpVersionNumber.Version20;

           request.SubscriberPreamble = "TEST";

           request.SubscriberCode = "1234567";

           request.SubscriberPassword = "ABC";

 

          BureauService service = request.AddService("Trended3D", "DXP1", "mySolutionId");

          // Trended3D does not require additional inputs - this is for demonstration purposes only

           service.AddInput(ApplicantCardinality.SubjectView, "SampleInput", "SampleName");

 

          return request.RequestProduct(connection, applicant);

       }

   }
