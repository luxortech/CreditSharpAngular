TotalViewEnhancers Class

 

Namespace: LuxorTech.CreditSharp.Connect

    

 

The static TotalViewEnhancers class is referenced when ordering Version 7.0 Equifax credit reports.  It is used to specify what extended data elements, if any, to include in the TotalView response. For instance, the month by month payment and status history that can be appended to each tradeline (known by Equifax as Dimensions) can be added to or excluded from each bureau response using this class.

 

By default, all enhanced responses are enabled.

 

Available enhanced TotalView elements:

 

Name

Content

Description

Addresses

Bool

Indicates if extended address segments are to be returned.

 

This corresponds to Optional Feature Code 'A' and increases the size of date fields from six characters (MMCCYY) to eight characters (MMDDCCYY).

Collections

Bool

Indicates if extended collection segments are to be returned.

 

This corresponds to Optional Feature Code 'K' and adds the increases the size of date and amount fields, as well including an account number field and refined product code detail.

Inquiries

Bool

Indicates if extended inquiry segments are to be returned.

 

This corresponds to Optional Feature Code 'K' and adds the AccountType field to the response.

PaymentPatterns

Bool

Indicates if payment patterns should be are to be returned in tradelines.

 

This corresponds to Optional Feature Code 'S'.

PublicRecords

Bool

Indicates if extended public record segments are to be returned.

 

This corresponds to Optional Feature Code 'K' and adds the increases the size of date and amount fields.

TradelineEnhancers

Enum

One of the EfxTradelineEnhancer enumerations.

Gets or sets the value indicating what type of extended tradeline detail is to be returned, if any.

 

EfxTradelineEnhancer Enumerations

Member

Value

Description

None

0

No extended fields returned.  

 

AllFields

1

Corresponds to optional feature code 'T'  (Default).

 

Returns all possible extended elements.

 

HighValueFields

2

Corresponds to optional feature code 'H'

 

Returns a subset of the extended fields offered with option feature code 'T' (AllFields).  For instance, the wider 9 digit balance and past due amounts found in AllFields are not returned.  (There is not much utility in requesting this option as the response size is exactly the same as the AllFields version where some items has been suppressed and blank filled.)

 

ExtendedFields

3

Corresponds to optional feature code 'N'

 

Adds additional narrative codes, scheduled payment amount, last payment date, wider dollar amounts and refined product code detail.

 

TermsPaymentFields

4

Corresponds to optional feature code 'R'

 

Adds just the actual and scheduled monthly payment amount as well as a refined number of payments value.

 

DimensionData

5

Corresponds to optional feature code 'G'

 

Returns all possible extended elements and includes up to 24 repeating groups of historical payment and status detail.

 

 

Either the AllFields or DimensionData enumerations are required for populating Historical Activity displays using the TradelineDimensions option in Html layouts.
