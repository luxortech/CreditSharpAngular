LuxorTech.CreditSharp.Connect Namespace

 

    

 

The Connect namespace is used to send requests for consumer credit information to each of the national credit repositories.  Currently the only communication protocol offered within CreditSharp is HTTP. At some point, based upon demand, support for the SNA or TCP/IP protocols may be introduced.  

 

The first step in initiating a dialog with one of the credit bureaus is to create a BureauConnection object.  This is where the general communication parameters are defined (the bureau's URL, any certificates, etc) and a communication channel is opened.

 

This connection is then used in conjunction with a bureau specific request object (EquifaxRequest, for example) to transmit an inquiry to the bureau and receive a credit data response.

 

An example of a typical programming pattern

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Connect;

 

  public class EquifaxConnect

   {

      public string OrderJoint(Applicant primary, Applicant secondary)

       {

          BureauConnection connection = new BureauConnection(BureauIds.Equifax);

           connection.HttpSettings.ServiceName = "ACROTEST";

           connection.HttpSettings.ServiceURL = "https://transport5.ec.equifax.com/servlet/stspost";

 

          EfxInquiryOptions options = new EfxInquiryOptions(licenseKey);

           options.ConsumerReferral = true;

           options.MilitaryBorrowerCheck = true;

 

          EquifaxRequest request = new EquifaxRequest(options);

           request.PermissiblePurposeCode = "01";

           request.ProductCode = EfxProductCode.CreditReport;

           request.ResponseVersion = EfxVersionNumber.Version50;

           request.SubscriberCode = "222XX11111";

           request.SubscriberPassword = "ABC";

           request.CpuResponse = EfxCpuResponse.FullFileFixed;

 

          string creditData = request.RequestProduct(connection, primary, secondary);

 

          return creditData;

       }

   }

Connect Classes

 

Name

Description

Core or general use classes

BureauConnection

Class containing settings and methods related to management of a communication session with a given bureau.

BureauScore

Class contains an instance of a bureau scoring model to be requested.

BureauSubscriber

Provides access to the output from a subscriber search request

HttpConfig

Container class for providing values related to internet destinations and permissions associated with credit bureau communications.

ProxyServer

Class used to specify settings when a proxy server is being used for bureau communications.

Equifax specific

EquifaxRequest

A class for preparing and executing an Equifax transaction.

EfxInquiryOptions

A group of properties specific to an Equifax credit bureau request.

Experian specific

ExperianRequest

A class for preparing and executing an Experian transaction.

ExpInquiryOptions

A group of properties specific to an Experian credit bureau request.

TransUnion specific

TransUnionRequest

A class for preparing and executing a TransUnion transaction.

TucInquiryOptions

A group of properties specific to a TransUnion credit bureau request.

TucLookup

Object used when performing a look up of subscriber contact detail.
