CommonOptions Class

 

Namespace: LuxorTech.CreditSharp.Datamart.Common

    

 

The CommonOptions class contains settings for influencing configurable CommonData operations.

 

Constructor

 

Name

Description

CommonOptions()

Constructor for the options class that must in turn be passed as a parameter to the CommonData constructor.

 

Syntax

public CommonOptions()

 

Parameters:  None

 

Example

 

using LuxorTech.CreditSharp.Datamart;

using LuxorTech.CreditSharp.Datamart.Common;

 

namespace HelpDemos

{

  public class DemoCommon

  {

      public CommonData NormalizeBureau(BureauData creditDataObject)

      {

          CommonOptions options = new CommonOptions();

          options.DateMask = "YYYYMM01";

          options.EstimateInstallmentPayments = true;

          options.EstimateRevolvingPayments = true;

 

          CommonData commonData = new CommonData(options);

          commonData.Normalize(creditDataObject);

          return commonData;

      }

   }

}

Content

Description

DateMask

Text (10)

Provides a date mask string indicating the order and positioning of Month, Day, Year when displayed along with the default day value.

 

The input will consist of a string with placeholders for where each date portion should appear in output fields, along with the desired delimiter (if any) desired between elements.

 

Given the input bureau date of March 13th, 2023, the example below show what each possible input mask would yield:

 

"MM/01/YYYY"

= 03/13/2023

"YYYY/MM/01"

= 2023/03/13

"YYYYMM01"

= 20230313

"MM-01-YYYY"

= 03-13-2023

"MM-01-YY"

= 03-13-23

 

As can be seen above, the Day portion of the input mask has been specified using a numeric two character value.  This not only serves to position the day portion of the date in the output string(s), but also provide a default value to be used when a Day element is not returned with a bureau date.

 

For example, given a bureau date of simply July, 2023, the following outputs can be generated:

 

"MM/15/YYYY"

= 07/15/2023   (A default day of 15 has been applied)

"YYYY/MM/01"

= 2023/07/01   (The more typical default day of '01' has been applied)

 

The DateMask property is initialized as: "MM/01/YYYY".

 

EstimateInstallmentPayments

Bool

Gets or sets the flag indicating whether the framework should calculate missing monthly installment payment amounts.

 

[Defaults to false]

EstimateRevolvingMinimum

Num (2)

Gets or sets the minimum dollar amount that can be assigned when estimating a revolving payment.

 

Monthly payments are calculated as a percentage of the current balance, but most creditors will have a minimum payment about established for lower balances.  This is typically $20, which is the default value used within CreditSharp.

[Defaults to 20]

EstimateRevolvingPayments

Bool

Gets or sets the flag indicating whether the framework should calculate missing monthly revolving payment amounts.

 

[Defaults to false]

EstimateRevolvingPercent

Dec

Gets or sets the percentage to be used in calculating missing monthly revolving payment amounts.

 

Monthly payments are calculated as a percentage of the current balance, with a widely used value of 1.5, which is the default value within CreditSharp.

 

[Defaults to 1.5]

Inherit

Bool

Gets or sets the flag indicating whether certain CommonData options should be inherited from those used when the active BureauData reference was itself created.

 

Inherited options are: DateMask and MaskSsn.

 

[Defaults to true]

MaskSsn

Char (1)

Gets or sets the character to be used as a mask for Social Security numbers.

 

If the property is left blank (the default), all social security data will appear unmasked, as it was received from the credit bureau.  Otherwise, the character provided here will replace the first five positions of each SSN field.

 

options.MaskSsn = "*";

 

With this setting, the SSN 666-99-6060 would be output as *****6060, or with dashes, ***-**-6060 if on a printed report.

 

Any printable character can be supplied here.

 

NumericIndicators

Bool

Gets or sets the flag indicating whether the ComTradeline and ComPublicRecord DeceasedIndicator, DerogatoryIndicator and DisputeIndicator fields are presented in their character or numeric forms.

 

[Defaults to true]

ZeroBasedMop

Bool

Get or sets the value which determines if a Tradeline Manner of Payment and Payment History codes will be numbered from zero or one.

 

When true, an instance of a current month would be represented as "0" while a 30 day late would be seen as "1", a 60 day late as "2", etc.  When false, the current month would be styled as "1" with 30 day lates being seen as "2" and 60 day lates being "3", etc.

 

Having the codes zero based allows for the explicit representation of a 180 day delinquent occurrence, as "6".  When the codes are based from one, then the value of "6" must stand in for either 150 or 180 days late.  

 

Please refer to the Mop Codes appendix topic for a full presentation of this optional feature.

 

[Defaults to false]
