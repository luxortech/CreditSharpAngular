Getting Started

 

    

 

All CreditSharp features are accessible through a single asset: the CreditSharp.dll assembly.  This assembly is built on the .NET 4.0 Framework and does not require any registry settings or additional installation steps.  Simply add a reference in your development project, spin your code and deploy it alongside your application.

 

The CreditSharp framework presents a number of targeted namespaces for use in processing credit bureau information. Beginning with types found in the Connect namespace, credit is requested and then passed on to the Datamart types, where a specific namespace is dedicated to each credit bureau. Once the bureau data has been imported it can be normalized and deduped using the Common and Merge types. Finally, the Analyze types are used to calculate aggregated values while the Format types facilitate generating various forms of outputs.

 

During instantiation, each root level CreditSharp component typically requires a reference to one or more prerequisite components. For example, the Datamart.ExperianData class requires that a BureauOptions object be created and populated so it can be provided during class construction.  The root objects only require the execution of a single method to accomplish the principal tasks for most typical production environments.

 

The basic sequence of operations, broken down into five steps, is highlighted below.

 

 

Step 1. Order credit report

[Connect]

The first stage of most production cycles will be to order consumer credit profiles from one (or more) of the three national credit bureaus. This is done using the functionality exposed within the Connect namespace.

Step 2. Import bureau data

[Equifax, Experian, TransUnion]

Once credit bureau data has been obtained, or has been provided, the next step is to import that data so that it is available for further processing and analysis. Step 3 and 4. Process and analyze

[Common, Merge, Analyze]

After a bureau response has been parsed it must be converted to a normalized common view in order to perform additional CreditSharp operations. Duplicate items can be removed using the Merge functionality while summary level data can be obtained using the Analyze functionality.

 

Namespace(s):

Key References:

LuxorTech.CreditSharp.Datamart.Merge

LuxorTech.CreditSharp.Analyze

Step 5. Retrieve results

[Format]

Once the data has been summarized (if desired) and normalized it can be extracted into several different formats, each serving different needs. There are methods for obtaining viewable reports in either Html or Text based layouts as well as generating Xml or Json data sets for use in additional computations in external systems.
