CreditSharp.dll.config file

 

    

 

The CreditSharp.dll.config utility can be used as a convenient means to apply a number of framework options through a single location instead of through individual programming instructions as well as supplying definitions for any custom score models that a given credit bureau might be delivering.

 

This configuration file does not need to be present for the CreditSharp framework to function properly. If the file is not located within the same folder as the assembly (CreditSharp.dll) then each option will be initialized to its default value.  

 

Using this configuration file allows you implement new CreditSharp features quickly without the need for modifying code or further customizing database structures (for example) to accommodate newly defined CreditSharp options.  A typical use case for the CreditSharp.dll.config might be when a customer always wants to use the same specific set of Format options to generate their reports. Rather than having your code execute the same instructions over and over again, the options can be specified just the one time in the configuration file. It also offers a easy mechanism for exploring how different outputs would appear without having to specifically code for each option being explored.

 

Settings that are assigned using the configuration file can always be overridden during runtime execution simply by assigning a different value using the property assignor for a given option.  The configuration file, in essence, just redefines / customizes the default values for each CreditSharp framework option.

 

In order for configuration file entries to take effect, the file needs to be placed in the same folder as the assembly file (CreditSharp.dll) and the file needs to be enabled by setting the csUseConfigSettings key to a value of "true".  The csUseConfigSettings key is near the bottom of the file. The default CreditSharp.dll.config file can be found in the Assets folder for the CreditSharp installation (typically C:\Program Files (x86)\Luxor Technologies, Inc\CreditSharp\Assets).  If only model definitions are being specified then the csUseConfigSettings key may be left as "false".

 

 

File contents:

 

The configuration file is an XML document which uses a series of name/value pairs to specify option values for all the various user controllable CreditSharp framework features.  The names and values of option settings mimic those found within the various option classes.  Links to enumerate and describe all the available settings can be found in this topic's parent chapter above.

 

All entries are case sensitive.  Boolean settings should be given values of "true" or "false".  Enumerated values are specified using the literal name and choice for that enum - the possible values can be found within the topic pages corresponding to the options class in question.  To select the internal default values for character or numeric based settings specify the value as "empty" otherwise supply a string literal for the desired setting.

 

Finally, settings that are either sensitive in nature or would be expected to change from one transaction to the next cannot be assigned using the configuration file.  As such, bureau credentials cannot be entered here (ie: your subscriber number or password) and consumer identifiers (name, social security number, etc) are not accessible here.

 

The document first outlines a section for each corresponding option class found within the framework using the same names with a prefix of "cs", within <configSection> nodes.  For example, the FormatOptionsHtml class settings can be found and applied within the config file inside the <configSection> with the name "csFormatHtmlOptions".  Following these, a series of <modelDefinition> segments can be entered.

When specifying custom score models, several key points should be followed and it should be noted that all entries are case-sensitive. When defining an Equifax model, the <modelDefinition> name attribute should be "csEfxScoreModel".  For Experian entries, use "csExpScoreModel" and for TransUnion entries use "csTucScoreModel".

 

Entries that will be used to look up model names or factor/error text should be entered to correspond with the form in which they will appear in the response data.  

 

With Equifax, codes for model id values are typically five characters long, right-justified and zero filled. Score factor codes are the same, typically five characters long, right-justified and zero filled.  For example, "09987" for a model id and "00010" for score factor 10, "00032" for score factor 32, etc.

 

With Experian, codes for model id values are two characters long and score factors are not decorated in any fashion (ie: not zero filled).  For example, you would see "C1" for a model id and "01" for score factor 01, "02" for score factor 02, etc.

 

With TransUnion, codes for model id values are five characters long, right-justified and zero filled.  Score factor codes are typically three digits, with one leading zero as necessary.  For example, you might see "00CX2" for a model id and "010" for score factor 10, "104" for score factor 104, etc.

 

Sample entries for each credit bureau are included at the end of the sample configuration file above.  Note that not all model definitions include Error codes or Adverse factor codes.  In those cases, it is perfectly acceptable to simply input the ModelId and ModelName key/values and omit the others.
