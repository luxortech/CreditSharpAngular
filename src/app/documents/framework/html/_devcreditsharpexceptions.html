CreditSharp Errors and Exceptions

 

    

 

Exception handling in CreditSharp employs a hybrid approach which makes use of both .Net style exceptions and traditional error codes.  Whenever an error is encountered, whether it is an invalid parameter or some sort of fatal system fault, the error state will be reported back to the caller by throwing a CreditSharp exception which has been derived from the Microsoft System.Exception class.

 

Unlike many .Net implementations, there is not a specific exception defined for every possible error event.  Instead, there are a set of basic exception classes generally corresponding to each primary operational region (ie: one exception class for reporting errors during Analyze operations and another for reporting errors from the Format namespace).  Within these, there is a Status property which contains an enumerated value indicating the specific error condition being reported.

 

Finally, an unique error code is also included in each CreditSharpException class which is modeled after the Microsoft HRESULT return codes.  A bit map is used to encode several pieces of error detail into a single numeric value. This EncodedStatus can be captured and sent to logs or displays as a more compact way of registering an error event.
