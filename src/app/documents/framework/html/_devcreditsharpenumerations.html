CreditSharp Enumerations

 

    

 

The following enumerations can be be found within the CreditSharp assembly:

 

Namespace

Description

LuxorTech.CreditSharp.Analyze

Aggregating credit data.

AnalysisActive

Enumerations available for use when invoking the ComTradeline IsActive method, which indicate if any specific date(s) should be ignored.

AnalysisCreditCapacity

Specifies how available limit calculation is performed.

AnalysisDate

Used to specify which date will be the basis for age related calculations.

AnalysisDateUnit

Specifies what unit is being measured when comparing the gap between two dates.

AnalysisId

Used to specify which collection of attributes will be / has been generated.

AnalysisLateSeverity

Specifies what level of delinquent occurrence is being evaluated.

AnalysisLimitProxy

Specifies if an alternate CreditLimit source can be used in relevant calculations.

AnalysisOperation

Indicates what type of instruction will be performed in a given evaluation.

AnalysisResultType

Indicates the fundamental data type of an evaluation result.

AnalysisTradeState

Specifies how the Open or Closed status of a given Tradeline is decided.

LuxorTech.CreditSharp.Connect

Communicate with credit bureaus.

ConnectionType

Indicates what communication protocol will be used.

ProxyType

Indicates if Http communications will be performed through a proxy server.

ScoreDisclosure

Used to specify if an Experian score disclosure form should be embedded within the credit bureau response.

ScoreSupplement

Indicates if supplemental consumer score disclosure elements are to be included in the bureau response.

LuxorTech.CreditSharp.Core

Shared definitions.

AddressRegion

Indicates which to which country or region an address belongs

ApplicantCardinality

Specifies which applicant's data is being presented or on which applicant's data an operation should apply.

ApplicantSupplementalIdType

A Enumerates the possible types of supplemental identification entries

BureauIds

Indicate which bureau (or bureaus) are currently being referenced.

BureauLocales

Indicate in which country the active bureau operates.

EfxVersionNumber

Indicates what Equifax response versions are available and/or recognized.

ExpVersionNumber

Indicates what Experian response versions are available and/or recognized.

OwnershipStatus

Indicates the financial relationship consumer has with given residential address.

Partitioning

Used to indicate how applicant data is displayed or aggregated.

PaymentBasis

Enumerates the possible ways in which an employee's salary my be computed.

TucVersionNumber

Indicates what TransUnion response versions are available and/or recognized.

LuxorTech.CreditSharp.Datamart

Bureau and Common data access.

ComIdentificationIds

Identifies elements within a Com/Mrg Identification segment for use in custom evaluations.

ComInquiryIds

Identifies elements within a Com/Mrg Inquiry segment for use in custom evaluations.

ComPublicRecordIds

Identifies elements within a Com/Mrg PublicRecord segment for use in custom evaluations.

ComTradelineIds

Identifies elements within a Com/Mrg Tradeline segment for use in custom evaluations.

LuxorTech.CreditSharp.Datamart.Common

Normalized credit data.

ComSegmentIds

Uniquely identifies a type of common data segment.

ConsumerPhoneContact

Indicates best times to use the associated phone number to contact the consumer.

ConsumerPhoneLocation

Indicates the physical location associated with phone number.

ConsumerPhoneOrigin

Indicates from which data segment the phone number was extracted.

ConsumerPhoneSource

Indicates from which database the phone number was extracted.

ConsumerPhoneType

Indicates what type of phone number is being reported.

LuxorTech.CreditSharp.Datamart.Merge

Merge data sets.

CombineMethod

Indicates how duplicate items are resolved once identified.

CompareMethod

Indicates how duplicate items between bureaus are identified.

MergeOperation

Indicates how what type of process, if any, will be applied to each data segment.

LuxorTech.CreditSharp.Datamart.Equifax

Equifax bureau responses.

EfxSegmentIds

Uniquely identifies the type of bureau segment, as shown in the DtSegmentId property of the BureauSegment class.

LuxorTech.CreditSharp.Datamart.Experian

Experian bureau responses.

ExpSegmentIds

Uniquely identifies the type of bureau segment, as shown in the DtSegmentId property of the BureauSegment class.

LuxorTech.CreditSharp.Datamart.TransUnion

TransUnion bureau responses.

TucSegmentIds

Uniquely identifies the type of bureau segment, as shown in the DtSegmentId property of the BureauSegment class.

TucServiceType

Indicates which basic TransUnion service provided the detail in a given segment.

LuxorTech.CreditSharp.Extract

Extract native bureau elements.

ExtractLayout

Indicates how extracted bureau data will be presented.

LuxorTech.CreditSharp.Format

Generate output.

FormatLayout

Indicates which output format is desired.

HtmlScoreRankDisplay

Specifies how a score's percentile rank will be decorated, if at all.

HtmlStyle

Indicates which report layout will be used when generating the Html output.

HtmlTradeStatusSource

Indicates how a tradeline status display is chosen in Html layouts.

HtmlTradeStatusDisplay

Indicates if a separate status element should be displayed in the tradeline section of an Html layout.

HtmlTradelineStyle

Specifies which tradeline layout will be employed in Html outputs.

JsonMergeHistory

Specifies if any merge history should be included in the JSON output.

JsonStyle

Used to indicate what JSON layout is being requested.

TextStyle

Indicates which report layout will be used when generating the Text output.

XmlMergeHistory

Specifies if any merge history should be included in the XML output.

XmlStyle

Used to indicate what XML layout is being requested.
