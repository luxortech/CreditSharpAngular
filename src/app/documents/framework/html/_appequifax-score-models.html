Equifax Score Models

 

    

 

The table below lists all Equifax models currently recognized by the CreditSharp framework.

 

Equifax allows for the return of certain score models using default customer accounts settings.  The mechanism for requesting these default models in CreditSharp is to supply a keyword for the score model, instead of a specific model code.

 

If a score model needs to be ordered that does not appear in this list, it can be preceded with an exclamation point to suppress the internal validation which prevents unknown models from being requested.  For example, EquifaxRequest.AddModel("!08872") would successfully append the hypothetical new model 08872 to the inquiry.

Keyword: CONFIG

 

In inquiries, this generates the optional feature code 'O' (for STS requests) or DefaultModelFlag 'Y' (for XML requests) which causes all the default scores to be returned.

 

To identify which scores are returned in responses, the model codes will either be prefixed with 'CP', as listed above, or have one of the numeric ids, as listed below.

 

Keyword: DAS

 

The Enhanced Delinquency Alert System scoring model will be requested with this keyword. The score model id will appear as 'DSE' in responses.

Keyword: LEGACY

 

Functionally equivalent to the FICO (or BEACON) keywords above, but instructs Equifax to return Fair Isaac score models using the older, legacy Beacon verbiage and model name(s).

 

In inquiries, this generates the optional feature code 'J' (for STS requests) or FICOFlag 'Y' (for XML requests) which causes the default FICO score to be returned.  Additionally, the XML tag UseFicoVerbiageFlag will be set to 'N'.
