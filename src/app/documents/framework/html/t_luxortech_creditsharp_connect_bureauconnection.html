BureauConnection Class

 

Namespace: LuxorTech.CreditSharp.Connect

    

 

The BureauConnection class contains the settings and methods related to management of a communication session with a given bureau.

 

A BureauConnection instance is one of the few CreditSharp object whose design allows it to be retained for more than one transaction.  If desired, a connection object can be created and reused in batch implementations where multiple transactions will be submitted to a given bureau in a single session.

 

If there will be some delay between each subsequent request, it is recommended that a new connection be established for each request.  This is because once a connection has gone idle with one of the bureaus, their host system will eventually terminate the session and any attempt to further communicate using the expired connection will generate an error.  The BureauConnection object is not designed to be re-entrant and will not reestablish a connection that has been terminated by the host system.

 

Constructors

 

Name

Description

BureauConnection(BureauIds bureauId)

Constructor for the BureauConnection class.

 

Defaults the HostSystem to BureauLocales.UnitedStates

BureauConnection(BureauIds bureauId, BureauLocales hostSystem)

Constructor for the BureauConnection class which also accepts the HostSystem parameter.

 

Note: Cross border credit requests between the United States and Canada can both be carried out using either system host.  Canadian consumer credit can be obtained through the US host systems and US consumer credit can be obtained through the Canada host systems.  The decision on which host to use will largely be dependent on a company's contract with the bureau's themselves.  If the originating service contract authorized bureau access with the United States credit bureau then that host system should be engaged, and likewise if the originating contract was through the Canadian credit agency then the Canada host system should be engaged.

 

The host system chosen to conduct the transaction dictates to what conventions the response data conforms.  Generally, requests routed through the US based hosts return data that is formatted to US bureau specifications while Canada host responses are formatted to the Canadian bureau specifications. Regardless of the host system being engaged CreditSharp will normalize the responses into the same common view.

 

 

Syntax

public BureauConnection(BureauIds bureauId)

public BureauConnection(BureauIds bureauId, BureauLocales hostSystem)

 

Parameters:

bureauId

Type: BureauIds enumeration

Indicates the credit bureau for which the connection is being created.

 

hostSystem

Type: BureauLocales enumeration

Indicates which host system will be handling communications.

Defaults to BureauLocales.UnitedStates if not provided.

 

Example 1. Default host

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Connect;

 

    public class EquifaxConnect

    {

        public string OrderJoint(Applicant primary, Applicant secondary)

        {

            BureauConnection connection = new BureauConnection(BureauIds.Equifax);

            connection.HttpSettings.ServiceName = "ACROTEST";

            connection.HttpSettings.ServiceURL = "https://transport5.ec.equifax.com/servlet/stspost";

 

            EfxInquiryOptions options = new EfxInquiryOptions(licenseKey);

            options.ConsumerReferral = true;

            options.MilitaryBorrowerCheck = true;

 

            EquifaxRequest request = new EquifaxRequest(options);

            request.PermissiblePurposeCode = "01";

            request.ProductCode = EfxProductCode.CreditReport;

            request.ResponseVersion = EfxVersionNumber.Version50;

            request.SubscriberCode = "222XX11111";

            request.SubscriberPassword = "ABC";

            request.CpuResponse = EfxCpuResponse.FullFileFixed;

 

            string creditData = request.RequestProduct(connection, primary, secondary);

 

            return creditData;

       }

   }

 

Example 2. Input host

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Connect;

 

    public class TransUnionConnect

    {

        public string OrderJoint(Applicant primary, Applicant secondary)

        {

            BureauConnection connection = new BureauConnection(BureauIds.TransUnion, BureauLocales.Canada);

            connection.HttpSettings.ServiceURL = "https://uat.tu.tuconline.com/xml/r4.asp";

 

            TucInquiryOptions options = new TucInquiryOptions(licenseKey);

 

            TransUnionRequest request = new TransUnionRequest(options);

            request.ProductCode = TucProductCode.CreditReport;

            request.ResponseVersion = TucVersionNumber.Version40;

            request.SubscriberCode = "9999999";

            request.SubscriberPassword = "ABC";

            request.SubscriberIndustry = "ZZ";

            request.XmlResponse = TucXmlResponse.Standard;

 

            string creditData = request.RequestProduct(connection, primary, secondary);

 

            return creditData;

       }

   }
