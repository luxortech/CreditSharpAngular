AddEvalution Method

 

Namespace: LuxorTech.CreditSharp.Analyze

    

 

The AddEvaluation method appends an attribute definition to a custom AnalysisBundle.  It is used to create a new attribute definition (evaluation) either for one of four credit data types or one that references other attributes (as opposed to directly inspecting a credit data response).

 

A brief usage example is show below.  A much more extensive example can be reviewed in the Customizing Analyze Attributes topic.

 

Syntax

public void AddEvaluation (EvaluateIdentification newEvaluation)

public void AddEvaluation (EvaluateInquiry newEvaluation)

public void AddEvaluation (EvaluatePublicRecord newEvaluation)

public void AddEvaluation (EvaluateReference newEvaluation)

public void AddEvaluation (EvaluateTradeline newEvaluation)

 

Parameters:

 

newEvaluation

Type: Data specific instance derived from the internal EvaluateBase abstract base class.

Each unique instance provides a placeholder for a custom attribute definition.

 

Exceptions

Exception class

Status

AnalyzeDataException

AnalyzeDataStatus.EvaluationAlreadyExists

 

An evaluation with the same name has already been added to the current bundle.

 

Example

 

          //

          // Statements can be defined using separate declarations of each lambda expression  

          //

          InquiryPredicate nbrAutoInqr2 = (inquiry, options) => inquiry.AGE2(ComInquiryIds.InquiryDate) < 15;

          bundle.AddEvaluation(new EvaluateInquiry("INQ_NbrAuto2", AnalysisOperation.Count, ComInquiryIds.IdCount, nbrAutoInqr2));

 

          //

          // Alternatively, statements can be supplied as inline lambda expressions directly when creating an evaluation  

          //

          bundle.AddEvaluation(new EvaluatePublicRecord("PUB_NbrLiens", AnalysisOperation.Count, ComPublicRecordIds.IdCount, (pubrec, options) => publicRecord.IsTaxLien));

 

          //

          // Lastly, statements can be supplied by referencing a compiled function to be invoked during processing

          //

          bundle.AddEvaluation(new EvaluateTradeline("TRL_HasBalance", AnalysisOperation.Count, ComTradelineIds.IdCount, (TradelinePredicate)TRL_HasBal));

 

          private static bool TRL_HasBalance(ComTradeline trade, AnalyzeOptions options) { return (!trade.AMT(ComTradelineIds.Balance) > 0); }
