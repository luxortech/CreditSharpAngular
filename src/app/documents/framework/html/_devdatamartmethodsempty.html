EMPTY Method

 

    

 

The EMPTY method is used to confirm whether or not a given data element is populated with an actual value.

 

Syntax

public bool EMPTY (enum T elementId)

 

Parameters:

 

name

Type: enum T (where elementId is an enumerated value specific to the Datamart type being evaluated)

 

The element to be tested.

 

Each data class within a CommonData (or MergeData) instance implements its own specific version of the EMPTY method where the input parameter is an enumerated type specific to the type of data.  For instance, the ComTradelineIds enumeration must be used to evaluate elements within a ComTradeline reference and ComInquiryIds would be used with a ComInquiry instance.

 

Example for creating custom attribute

 

Setup a custom attribute definition to count tradelines which have a Credit Limit amount

 

using LuxorTech.CreditSharp.Analyze;

using LuxorTech.CreditSharp.Datamart.Common;

 

namespace HelpDemos

{

  public class DemoAnalyze

  {

      public string CreateAttribute(AnalysisBundle bundle)

      {

          //

          // The following represents the creation of a custom analyze attribute

          //

          TradelinePredicate haveLimit = (trade, options) => !trade.EMPTY(ComTradelineIds.CreditLimitAmount);

          EvaluateTradeline nbrWithLimt = new EvaluateTradeline("NbrWithLimit", AnalysisOperation.Count, ComTradelineIds.IdCount, haveLimit);

          bundle.AddEvaluation(nbrWithLimt);

 

      }

   }

}
