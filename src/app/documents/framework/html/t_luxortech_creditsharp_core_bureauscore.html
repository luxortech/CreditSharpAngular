BureauScore Class

 

    

 

The BureauScore class is used to define the properties of a bureau scoring model that is not already included in the CreditSharp Framework's model definitions.  This typically would be used to include a score model that is unique (custom) to a specific bureau customer.

 

The class is not publicly creatable but an instance is returned when invoking the AddModel() method on one of the credit bureau data objects (EquifaxData, ExperianData or TransUnionData).  Once a class reference has been successfully created, then any error codes and/or adverse factor codes can be added to the new model definition.

 

When defining values that will be used to look up model names or text, be sure to enter the corresponding codes are in the same form as they will appear in the response data.  

 

With Equifax, model id values are typically five characters long, right-justified and zero filled. Score factor codes are the same, typically five characters long, right-justified and zero filled.  For example, "09987" for a model id and "00010" for score factor 10, "00032" for score factor 32, etc.

 

With Experian, model id values are two characters long and score factors are not decorated in any fashion (ie: not zero filled).  For example, you would see "C1" for a model id and "01" for score factor 01, "02" for score factor 02, etc.

 

With TransUnion, model id values are five characters long, right-justified and zero filled.  Score factor codes are typically three digits, with one leading zero as necessary.  For example, you might see "00CX2" for a model id and "010" for score factor 10, "104" for score factor 104, etc.

 

Not all model definitions include Error codes or Adverse factor codes.  In those cases, it is perfectly acceptable to simply call the AddModel() method to assign a name to the custom model.

 

Equifax Example

 

using LuxorTech.CreditSharp.Core;

using LuxorTech.CreditSharp.Datamart;

using LuxorTech.CreditSharp.Datamart.Equifax;

using LuxorTech.CreditSharp.Datamart.Common;

using LuxorTech.CreditSharp.Format;

 

namespace HelpDemos

{

  public class DemoCustom

  {

        public string EquifaxCustom(string creditDataBuffer)

        {

            EquifaxData equifaxData = new EquifaxData(new BureauOptions("licenseKey"));

            BureauModel customModel = equifaxData.AddModel("CM1", "Custom Model One");

            customModel.AddFactor("01", "Adverse factor code number one");

            customModel.AddFactor("02", "Adverse factor code number two");

            customModel.AddFactor("03", "Adverse factor code number three");

            customModel.AddFactor("04", "Adverse factor code number four");

 

            equifaxData.Parse(creditDataBuffer);

 

            CommonData commonData = new CommonData(new CommonOptions());

            commonData.Normalize(equifaxData);

 

            FormatData formatData = new FormatData(new FormatOtions(FormatLayout.Html));

            return formatData.Generate(commonData);

        }

  }

}
